## 情景 A：第一次接触电赛

**30 分钟流程**：

```
0~5 分钟  → 读 [README.md](../README.md) 了解仓库布局
5~15 分钟 → 读 [备赛系统/B_历年真题实战/_第一梯队总览.md](B_历年真题实战/_第一梯队总览.md)
            搞清楚：什么是 S 级题、A 级题
            决策：你想往"控制"/"仪表"/"电源"哪个方向走
15~25 分钟 → 打开你方向的"00_方向总览与训练计划.md"，浏览训练路径
25~30 分钟 → 选 1 个最简单的基础电路（如 [TB6612 电机驱动](A_方向训练/控制与智能车/01_基础电路库/电机驱动_TB6612.md) 或 [Buck 降压电路](A_方向训练/电源与电力电子/01_基础电路库/Buck降压电路.md)）
            动手开始
```

**第一周训练目标**：跑通 1 个基础电路 + 学会用 STM32CubeIDE 烧录

## 情景 B：有 MCU 基础但没打过电赛

**30 分钟流程**：

```
0~10 分钟 → 读 [_第一梯队总览.md](B_历年真题实战/_第一梯队总览.md) 选 1 道 S 级题
            推荐：第一次刷 = 2024 H 自动行驶小车（最容易跑通）
10~20 分钟 → 打开你选的题的 `05_完整复刻指南.md`
            按"第 1 周买什么 / 第 2 周搭硬件"清单照抄
20~25 分钟 → 看引脚分配表 + 接线表
25~30 分钟 → 在 GitHub / 淘宝下单 BOM 里的元件
```

**第一题目标**：4 周内跑通 2024H 任务 1（A→B 直线 1m）

## 情景 C：打过电赛但成绩一般

**30 分钟流程**：

```
0~5 分钟  → 读 [评委隐性评分细则.md](E_赛场实战/评委隐性评分细则.md)
            认知校准：你以前丢分丢在哪里
5~15 分钟 → 读 [_5题FAQ高频踩坑.md](B_历年真题实战/_5题FAQ高频踩坑.md)
            找你以前踩过的坑 → 看这里怎么解
15~25 分钟 → 读 [赛中4天每小时排程.md](E_赛场实战/赛中4天每小时排程.md)
            对照你上次比赛的节奏，找时间分配错的地方
25~30 分钟 → 选 1 道近 5 年没做过的真题，打开 `00_深度审题与方案论证.md`
            把它当作"复盘 + 提升"的训练
```

**3 周训练目标**：把上次比赛丢分的板块系统补一遍

---

## 情景 D：现在就要做某届某题

**30 分钟流程**：

```
0~3 分钟  → cd 到题目目录：备赛系统/B_历年真题实战/<年份>/<题号>/
3~10 分钟 → 顺序读：00_深度审题 → 00_做题指南 → 05_完整复刻指南（如有）
10~20 分钟 → 看：01_代码/引脚分配表 + 02_硬件/接线表 + 02_硬件/万用板布局图
            （A 级题没有万用板布局图，自己画一张）
20~25 分钟 → 复制代码骨架到 IDE：
             STM32 题 → 参考 [STM32G4 工程骨架](C_通用能力/STM32开发/STM32G4_工程骨架.md)
             MSPM0 题 → 参考 [MSPM0G3507 工程骨架](C_通用能力/TI_MSPM0开发/MSPM0G3507_工程骨架.md)
25~30 分钟 → 改 config.h 里的关键参数，开始写第一个 main.c
```
