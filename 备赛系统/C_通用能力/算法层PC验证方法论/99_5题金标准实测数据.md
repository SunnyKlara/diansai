### V1 → V2 优化记录（2026-05-26）

V1 用固定 512 点采样 → 频谱泄漏导致 3.5% 误差  
V2 自适应整周期采样：找最接近 512 的整数周期长度 → 0 频谱泄漏

```python
m_cycles = max(1, round(n_samples * freq / fs))
n_actual = round(m_cycles * fs / freq)
```

**4 行代码提升精度 25×**。这是 PC 仿真带来的最大复利。

### 调参留下的工程问题（让现场知道要处理）

PC 仿真还暴露了 1.9~2.3V 的超调，**这就是 PI 软启动要解决的现场问题**。方法论指导：
- 加 ramp-up：让 setpoint 在 100ms 内从 0 缓增到目标值
- 加 anti-windup：积分饱和时停止累积
- 限制 d(iref)/dt：电流参考变化率限幅

→ 这些参数都先在 PC 上调，再上 MCU。
