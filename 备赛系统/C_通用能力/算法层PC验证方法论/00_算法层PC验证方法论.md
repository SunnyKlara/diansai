# 算法层 PC 验证方法论 —— 电赛刷题的核心打法

> 这是从 5 道真题（2024B、2024H、2025A、2025G、2021A）实战中抽出来的方法论。
> 把所有"算法逻辑"先在 PC 上跑通到 0 误差，再去碰硬件 —— 这是国一队伍和普通队伍的最大区别。

---

## 一、为什么这是电赛的核心打法

### 普通队伍 vs 国一队伍的工作流对比

| 阶段 | 普通队伍 | 国一队伍 |
|---|---|---|
| 写算法 | 直接在 MCU 上写 + 烧 + 调 | **先在 PC 上写 Python 等价实现 + 验证 0 误差** |
| 调精度 | 改一个参数 → 烧录 → 测 → 反复（10min/轮） | PC 上跑 Python 仿真（1s/轮）+ 边界条件全覆盖 |
| 出问题 | 不知道是算法 bug 还是硬件 bug | UART DUMP 原始数据 → PC 跑金标准 → 30s 定位 |
| 现场调参 | 蒙参数 → 改 → 烧 → 看现象 | PC 仿真锁定参数范围 → MCU 直接收敛 |

**单次调试速度差 10×，调参覆盖率差 100×**。

### 这套打法的三个核心信条

1. **算法 0 系统性误差是最低标准** —— 容差全部留给硬件链路
2. **金标准 = 长期资产** —— 每改一行算法都自动验证回归
3. **任何硬件读数异常 30 秒内能判断"算法 vs 硬件"**

### 验证三层金字塔

每个算法必须通过这三层才算就绪：

```
        ┌─ Layer 3: 端到端冒烟（合成 ADC 原始数据 → cal_helper → 复现答案）
       /
      ┌─ Layer 2: 闭环仿真（输入 → 算法 → 输出 vs 解析值）
     /
    ┌─ Layer 1: 单元测试（已知答案的合成信号 → 算法 → 0 误差）
```

## 三、5 个真题的金标准产出（参考用）

### 2024 B 单相功率分析仪
**算法**：整周期采样 + 单频 DFT × 10 谐波  
**理论误差**：0.0000%（受限于双精度浮点）  
**关键洞察**：fs × N / freq 必须整数才无频谱泄漏

### 2024 H 自动行驶小车
**算法**：PID（速度 / 航向 / 循迹 / 转向）+ 红外加权位置 + 路径段管理  
**仿真结果**：1m 直线 1.14s（限 15s），终点偏 6.2mm  
**关键洞察**：差速运动学 PC 仿真能预测 80% 现场行为

### 2025 A 能量回馈变流器
**算法**：SVPWM 七段式 + RMS 滑动窗口 + 电压外环 PI  
**理论精度**：线电压 RMS 100% 等于 m × Vdc / √2  
**关键洞察**：48V 母线 + m_max=0.95 撞顶达不到 32V → **必须用 53V**

### 2025 G 电路模型探究装置
**算法**：扫频 + 单频 DFT 提取增益相位 + FFT 数字孪生  
**仿真结果**：增益误差 0.035 / 相位误差 0.027 rad（容差 0.10 / 0.17）  
**关键洞察**：FFT 推理路径在 PC 验证后再上 STM32 + CMSIS-DSP

### 2021 A 信号失真度测量装置
**算法**：过零检测 + 同步采样 + FFT + THD  
**仿真结果**：5/10/20/30/50% THD 全部误差 < 0.005%  
**关键洞察**：同步采样下用矩形窗，非同步用平顶窗 + ±2 bin 取最大

## 四、新题的标准刷题流程（按这个顺序做不会错）

```
Day 0：拿到题目
  ↓
1. 复制 _template/ 到 <题号_题名>/
  ↓
2. 写 00_深度审题与方案论证.md（5 视角分析 + 误差预算 + 功耗预算）
  ↓
3. 拆 Algorithm/ 模块 + 定义头文件接口（先有接口再有实现）
  ↓
4. 写 algo_reference.py（与头文件接口 1:1 等价的 Python 实现）
  ↓
5. 跑通算法层 PC 验证（理论精度达标）← 这一步通过才能上 MCU
  ↓
6. 写 Drivers/（含 calibration + uart_dbg 调试通道）
  ↓
7. 写 main.c 状态机（含 LPM）
  ↓
8. PC 端写 cal_helper.py（校准 + 数据导出）
  ↓
9. MCU 移植 → UART 导出原始数据 → cal_helper.verify → 与 algo_reference 对比
  ↓
10. 校准互感器 / 传感器 → 现场实测精度
```

**关键节点（不达标不进入下一步）**：
- 步骤 5 不达标：算法本身有 bug，到了 MCU 也是错的
- 步骤 9 不一致：MCU 端实现有 bug，必须先修复

## 六、反模式（千万别做）

❌ 拿到题就在 MCU 上写代码（80% 时间会浪费在调外设上）
❌ 不写金标准（出 bug 时分不清是算法还是硬件）
❌ Python 和 C 用不同算法（金标准就失效了，必须 1:1）
❌ 算法层和驱动层混写（算法层应该可单元测试）
❌ 现场临时改采样率（破坏整周期采样 → 频谱泄漏）
