# 选题决策流程（比赛第1天上午用）

> 拿到题目后的前2小时决定成败

## 决策流程图

```
拿到题目（通常6~8道）
    │
    ▼
全队快速浏览所有题目（30分钟）
    │
    ▼
排除明显不擅长的题目
    │
    ▼
剩余2~3道题深入分析（30分钟/题）
    │
    ├── 分析1：基本要求能不能做完？（最重要）
    ├── 分析2：手头有没有现成的硬件/代码？
    ├── 分析3：发挥部分能做几项？
    └── 分析4：报告好不好写？
    │
    ▼
打分比较，选定题目（不超过开赛后2小时）
    │
    ▼
确定方案，开始分工
```

## 选题评分表

对每道候选题打分（1~5分）：

| 评估维度 | 权重 | 题目A | 题目B | 题目C |
|----------|------|-------|-------|-------|
| 基本要求有把握完成 | ×3 | | | |
| 手头有现成硬件/模块 | ×2 | | | |
| 有相关代码/经验 | ×2 | | | |
| 发挥部分能做1项以上 | ×2 | | | |
| 报告容易写 | ×1 | | | |
| **加权总分** | | | | |

**选总分最高的那道题。**

## 关键原则

1. **基本要求能做完是底线**。基本要求做不完=白忙活
2. **不要被发挥部分的高分诱惑**。发挥部分做不出来很正常
3. **选自己最熟悉的方向**。比赛不是学习的时候
4. **2小时内必须决定**。犹豫不决是最大的浪费
5. **一旦选定不要换题**。换题=重新开始=必输

## 各方向快速判断清单

### 看到电源题（A题）
- [ ] 我们有全桥/半桥逆变的经验吗？
- [ ] 我们有MOSFET驱动板吗？
- [ ] 我们会SPWM/SVPWM吗？
- [ ] 我们有功率分析仪可用吗？
→ 全部打勾就选

### 看到仪表题
- [ ] 我们会FFT吗？
- [ ] 我们有高速ADC模块吗？
- [ ] 我们会信号调理电路吗？
→ 全部打勾就选

### 看到智能车题
- [ ] 我们有小车底盘吗？
- [ ] 我们会循迹PID吗？
- [ ] 我们有OpenMV吗？
→ 全部打勾就选

### 看到无人机题
- [ ] 我们有飞行器和飞控吗？
- [ ] 我们有飞行调试经验吗？
- [ ] 我们有视觉识别方案吗？
→ 全部打勾就选，否则**绝对不要选**
