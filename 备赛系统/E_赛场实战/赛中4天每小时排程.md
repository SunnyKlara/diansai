# 赛中 4 天每小时排程（小时级精确）

> 比 `4天时间分配模板.md` 更细：精确到小时 + 关键里程碑 + Go/No-Go 检查点  
> 三人队伍：硬件 H / 软件 S / 报告兼测试 R  
> **绝对原则**：**Day3 中午前必须跑通基本要求**，否则停下来全队救火

---

## Day 0（周二，赛前一天）

不要熬夜！按 [赛前24h终检清单.md](赛前24h终检清单.md) 自检，22:30 前必睡。

---

## Day 1（周三）

### 08:00 — 09:00 选题（全员）
- [ ] 全队读完所有题（每人 5 分钟扫一遍）
- [ ] 按 [选题决策流程.md](选题决策流程.md) 8 维评分
- [ ] **09:00 整必须出结果**，不允许犹豫超过 1 小时
- [ ] 选定后立刻报组委会（如有）

### 09:00 — 11:00 审题与方案（全员）
- [ ] 用 [00_深度审题与方案论证.md 模板](../B_历年真题实战/_深度审题模板.md) 写 1.5 小时
- [ ] 5 分项指标分解：每个发挥项映射到一个硬件/软件子任务
- [ ] 关键算法 → 选定（FFT vs DFT、PID vs LQR、SVPWM vs SPWM）
- [ ] **指标硬性约束写到白板**（精度 1%、稳态 0.2%、瞬态 100ms、损耗 < 5W）

### 11:00 — 12:00 分工 + 开始
- H：列硬件 BOM（按已有 BOM 模板套），核对学校实验室库存
- S：搭代码骨架（从最相近 S 级题骨架 git clone）
- R：起草设计报告**第 1 章绪论 + 第 2 章方案选择**（10:00 成型）

### 12:00 — 13:00 午餐 + 短休（强制 30min）

### 13:00 — 18:00 主线推进
- H：焊接 **电源板**（最优先，没电源啥都做不了）→ 17:00 完成
- S：完成 **Drivers 层**（PWM / ADC / UART / OLED）→ 17:00 板子点亮
- R：补**第 3 章理论分析**（计算公式 + 选型论证）

### 18:00 — 18:30 晚餐

### 18:30 — 22:00 主板焊 + 烧录联调
- H：焊**主控板 + 信号板**
- S：写 **Algorithm 层**（PID / 控制律 / 状态机）
- R：跟班焊接 + 写**第 4 章硬件框图**

### 22:00 检查点 1（Go/No-Go）
- [ ] 主控可烧录 + UART 出 hello world
- [ ] 电源板 5V/3.3V 输出正确
- [ ] OLED 能显示静态字符
- [ ] 一个 PWM 通道有波形（用示波器看）

**❌ 4 项不全 → 22:00~23:30 强制聚焦填坑，23:30 全员睡**  
**✅ 4 项全过 → 23:00 睡**

### Day 1 结束目标
- 硬件：主板焊完，所有模块焊完，电源系统 OK
- 软件：所有 Drivers 层写完，至少一个能跑（PWM 或 ADC）
- 报告：3000 字（绪论 + 方案 + 理论 + 硬件框图）

## Day 2（周四）

### 08:00 — 09:00 早会 + 上电检查
- [ ] 重新上电整板，确认昨天的 4 项检查点仍 OK
- [ ] 软件清单核对：今天必须实现哪 5 个核心函数

### 09:00 — 12:00 联调主线
- H：焊接 **传感器接口 + 调理电路**（精度链路）
- S：搭 **闭环控制**（PID 闭环跑起来，先空载）
- R：用 [测试数据记录模板.md](测试数据记录模板.md) 准备测试表

### 12:00 — 12:30 午餐 + 5 分钟短同步会

### 12:30 — 14:00 闭环跑通
- 把传感器 → ADC → 算法 → PWM 输出闭环串通
- **里程碑**：能看到输出值跟随设定值（不要求精度，先动起来）

### 14:00 检查点 2（Go/No-Go）
- [ ] 闭环能跟随设定值（不要求精度）
- [ ] OLED 能显示实时数据
- [ ] UART 调试通道可用（PC 端 cal_helper.py 能跟装置对话）

**❌ 任一项不过 → 立刻全员救火，不再分工**

### 14:00 — 18:00 调精度
- 用 [PID实战调参表.md](../B_历年真题实战/_PID实战调参表.md) 起步值开始调
- 每改一次参数 → R 同步记录到测试数据表
- **18:00 前必须达到基本要求 50% 精度**

### 18:00 — 18:30 晚餐

### 18:30 — 22:00 基本要求冲刺
- 把所有基本要求项都过一遍
- R 开始写**第 5 章软件设计 + 第 6 章测试方法**
- H 把所有杜邦线**用扎带固定**（防比赛震动松脱）

### 22:00 检查点 3（Go/No-Go）
- [ ] **基本要求 100% 跑通**（每一项都演示一遍）
- [ ] 所有指标用 [测试数据记录模板.md] 测过 3 次
- [ ] 报告达到 6000 字（绪论 + 方案 + 理论 + 硬件 + 软件 + 测试）

**❌ 基本要求未通 → 23:00 前果断启用 [赛场降级清单.md](../B_历年真题实战/_赛场降级清单.md)**  
- 若降级后仍不通 → 23:30 全队**休息 5 小时**（精疲力尽继续干只会出错）

### Day 2 结束目标
- 基本要求全部跑通
- 报告 6000 字
- 测试数据 3 组以上

### 08:00 — 09:00 早会 + 重测基本要求
- [ ] 上电后**立刻全部基本要求重测一遍**（夜间元件可能漂移）
- [ ] 数据稳定再开始发挥

### 09:00 — 12:00 发挥项 1（最易拿分的）

- S：编写发挥项算法
- H：焊接发挥项需要的额外硬件
- R：测试 + 记录

### 12:00 检查点 4（Go/No-Go）
- [ ] 第 1 个发挥项完成
- [ ] 测试数据 3 组以上

### 12:00 — 13:00 午餐

### 13:00 — 18:00 发挥项 2 + 报告冲刺
- 同样流程做第 2 个发挥项
- R 把**第 7 章测试结果**填满（含图表 + 误差分析）

### 18:00 — 18:30 晚餐

### 18:30 — 22:00 发挥项 3 + 报告完善
- 第 3 个发挥项（如果时间允许）
- 报告冲到 9000 字（加完整测试结果 + 误差分析 + 总结）

### 22:00 检查点 5（Go/No-Go）

**❌ 不达标 → 接受现状，转去优化报告 + 整理测试数据**

### Day 3 结束目标

- 报告 9000 字

- [ ] 万用板螺丝紧固

### 16:00 — 17:00 提交（如有具体提交时间窗）

## Anti-pattern（4 大忌）

1. **Day 1 末仍在选题**——选题时间死线 9:00 就是 9:00
2. **Day 2 末不去做基本要求**——基本要求是底线，先保证它通过再做发挥
3. **Day 3 才动报告**——报告要从 Day1 起每天写，不要堆到最后
4. **Day 4 改硬件**——只允许调软件 / 改报告，禁止焊接（焊错就完了）
