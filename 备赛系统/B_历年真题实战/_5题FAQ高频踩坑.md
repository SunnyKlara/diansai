## 通用 FAQ（所有 5 题都可能遇到）

### Q1：上电瞬间装置炸 / 烧管 / 冒烟
**根因**：
1. PWM 默认非 0（复位电平不可控）
2. 方向引脚悬空
3. 母线电容反接 / 极性错
4. STBY / MOE 默认低导致直通

**对策**：
- `Init()` 函数**第一行**写 PWM=0、所有方向 GPIO=0
- 母线电容**先用万用表确认极性**再焊
- TB6612 STBY 上电后软件**立即拉高**
- 电源**先用低压（12V）测试**，再升到额定

### Q2：编译报错 `undefined reference to ...`
**根因**：
1. SysConfig / CubeMX 没 Generate
2. 文件没加入工程
3. include 路径不对

**对策**：
- CubeMX/SysConfig 改完一定 Generate 一遍重新编译
- 自己加的 .c 文件确认在 `Project Tree` 里
- include 路径加 `01_代码/Drivers` 和 `01_代码/Algorithm`

### Q3：调试器烧不进去
**根因**：
1. 板子没上电（USB 供电不够）
2. 上次软件死循环了 → 长按 RST + 上电再松
3. SWD 引脚被复用了

**对策**：
- 板载 LED 亮才算上电
- 长按 RST 进 ISP 模式再烧
- 检查 SysConfig 是否把 PA13/PA14（SWD）配成普通 GPIO

### Q4：串口看不到任何输出
**根因**：
1. 串口号选错（设备管理器看 XDS110/ST-Link COM）
2. 波特率不匹配（115200 vs 921600）
3. printf 没重定向到 UART
4. 共地缺失

**对策**：
- 设备管理器找 COM 号
- 波特率 115200 是默认，不要乱改
- 加 `__io_putchar` 重定向 printf
- 万用表通断挡测共地

### Q5：电池电压突然掉
**根因**：
1. 锂电池保护板触发（短路 / 过流）
2. 电量真的低了
3. 接触不良

**对策**：
- 18650 装电池盒前先充满（4.2V）
- 加电压表实时监测
- 比赛前 30 分钟充电
- **比赛包必带备用电池**

### Q6：输出值乱跳 / 噪声大
**根因**：
1. 共地缺失
2. 电源纹波
3. ADC 采样时间太短
4. 模拟信号没去耦

**对策**：
- 数字 / 模拟 / 功率三地分开后**单点共地**
- 母线加 470μF + 0.1μF 去耦
- ADC 采样时间 ≥ 8 个 ADC 时钟
- ADC 输入加 100Ω + 1nF 低通

### Q7：调试器突然连不上
**根因**：
1. 板子被 SD-MMU 死锁
2. 上次代码进了 BootROM

**对策**：
- 长按 RST + 上电 + 持续 5s 再松（强制 BootROM）
- 在 IDE 里点 "Connect under Reset"
- 实在不行：刷 PC13 板载按键 + 上电

### Q8：现场环境与训练场差异大
**根因**：
1. 光线差异（IR 阈值漂）
2. 温度差异（IMU 零偏漂）
3. 振动差异（机械松动）

**对策**：
- **每次比赛前必做**：现场重校 IR / IMU / ADC 偏置
- 装置加机械减震
- IR 阈值用迟滞窗（不是单一阈值）

## 2024 H 自动行驶小车 专属

### Q：循迹时一直右偏
看 `04_调试记录/故障决策树.md` 第 3.2 节。最常见：左右轮速度差。

### Q：弯道飞出
- KP 不够 → 增加 KP × 1.5
- 速度太快 → 弯道前减速
- 看 IR 是否丢线（白色背景应有信号）

### Q：编码器只一边有数
TIM3 / TIM4 都要在 SysConfig 配 Quadrature Encoder Mode。

### Q：MPU6050 yaw 漂移
- 启动校零（IMU_Calibrate 必须 < 5°/s）
- ZUPT（静止强制 ω=0）
- 长时间任务用 IR 修正绝对位置

## 2024 B 单相功率分析仪 专属

### Q：DFT 测出 H3 比理论值偏低 5%+
- 整周期采样校验：fs / freq = 整数？
- 加汉宁窗 + 校正系数 ×2.0
- 取 ±1 邻居 bin 最大值

### Q：LCD 一片黑
- HT1621 初始化序列正确？
- CS / WR / DATA 三线焊接？
- 电源 3.3V 稳？

### Q：LPM3 进不去
- 中断没 `__bic_SR_register_on_exit`
- I/O 引脚悬空耗电
- 主循环必须以 `__bis_SR_register(LPM3_bits | GIE)` 结尾

### Q：电流测量精度 < 1% 但电压不行
- ZMPT101B 比 ZMCT103C 难调（电压互感器漏感大）
- 加 OPA2277 高输入阻抗缓冲
- 实测对比标准源校准

## 2025 G 电路模型探究 专属

### Q：100Ω 标准电阻测出 110Ω
- 取样电阻必须 0.1% 精密
- 4 线开尔文消除接触电阻
- 启动 ADC 自校零

### Q：DDS 高频段失真
- AD9833 输出加 LC 低通 fc=100kHz
- 减小 DDS 输出幅度（防 OPA 失真）
- 换 OPA2350（GBW=4MHz）替代 OPA2277

### Q：双 ADC 数据偏差大
- 必须 Regular Simultaneous 模式
- ADC2 不需要单独 DMA
- 读 ADC1->DR 拿 32 位（高 16 位是 ADC2）

### Q：X-Y 模式画的不是预期形状
- DAC 两通道时钟必须同步
- Y 与 X 处理逻辑：实部 → X, 虚部 → Y
- 示波器中点偏置 1.65V

## 2023 A 单相逆变器并联 专属

### Q：上电烧 4 个 MOSFET
**100% 是死区设错或自举电容问题**。
- DTG = 0x88（800ns）实测确认
- 自举电容用 1μF CBB **不要电解**
- 先 12V 上电 → 18V → 24V → 36V 阶梯升

### Q：母线预充失败
- 50Ω 预充电阻烧了？换 50Ω/10W
- 继电器响应？测 PB10 输出
- 软件超时设 5000ms 不要 1000ms

### Q：THD > 2%
- 死区减到 600ns（保留余量）
- 调制比降到 0.92（不要 0.95）
- 加死区电压补偿

### Q：双机并联环流大
- 频率靠晶振一致（HSE ±20ppm）
- 输出端各串 0.5mH 电感增加阻抗
- 工业级方案：PLL 锁相 + Droop 控制

## 2025 A 能量回馈变流器 专属

### Q：上电烧 6 个 MOSFET（一次性全死）
**血泪教训：直接 36V 上电的后果**。
- 必须 12V 试 → 18V → 24V → 36V
- 死区 500ns 严格确认（DTG = 0x55）
- 调压器渐进升压

### Q：三相对称度差
- HRTIM TIM_A/B/C 周期一致？都 = 5440000/20000
- 三个相 IR2110 同批次（容差 ±5%）
- DC 母线纹波必须 < 1%

### Q：同步整流不工作
- 下管 PWM 时序 = 上管反相 + 早 200ns 关
- HRTIM DTG 不能为 0
- 用比较器监测（实测电流方向）

### Q：dq 锁相失败
- SRF-PLL Kp/Ki 起步 100/5000
- 看 vq 应稳定在 0
- 频率漂动时验证锁相能跟 ±0.5Hz

## A 级 5 题快速踩坑

### 2021 F 智能送药
- OpenMV 数字识别误判 → 多帧投票（连续 3 帧一致）
- NRF24 双车地址写错 → 一台 RX=0xCAR1, TX=0xCAR2，另一台反过来

### 2022 C 小车跟随
- HC-SR04 ECHO 5V 直接接 MCU 烧管 → 必须分压到 3.3V
- 跟随车启动后退烧管 → 强制等 1s 再开 PID

### 2023 E 运动追踪
- 舵机用 MCU 3.3V → 电流不够，必须独立 5V/3A 适配器
- A4 旋转角识别失败 → 改用拨码手动设置

### 2025 E 自瞄
- 双 MCU 不共地 → UART 数据全错
- OpenMV 启动慢 30s → 先上电再动小车

### 2021 G 飞行器
- 自研飞控翻车 → 必须用 Pixhawk + PX4
- LoRa 与 GPS 互扰 → 天线远离 30cm 以上
