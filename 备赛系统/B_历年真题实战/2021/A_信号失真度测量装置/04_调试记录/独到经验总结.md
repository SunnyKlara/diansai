# 2021A THD测量 —— 独到经验总结

## 一、这道题的隐藏难点

### 表面难点 vs 真正难点

| 表面难点 | 真正难点 |
|----------|----------|
| FFT算法实现 | 频谱泄漏导致幅度不准 |
| 30mV小信号放大 | 放大器的噪声底限制了THD测量下限 |
| 100kHz高频采样 | TI片内ADC速率限制（MSP432只有1MSPS） |
| 手机蓝牙显示 | 10秒时间限制内完成所有操作 |

### 最容易丢分的地方

**误差Δ=THDx-THDo≤5%（基本要求）/ ≤3%（发挥部分）**

这个误差是**绝对误差**，不是相对误差！
- 如果真实THD=10%，测量值必须在5%~15%之间（基本）或7%~13%之间（发挥）
- 如果真实THD=50%，测量值必须在45%~55%之间
- 看起来不难，但频谱泄漏可以轻松导致5%以上的误差

## 二、信号调理的深层思考

### 30mV信号的放大器选型

30mV峰峰值 = 10.6mVrms，需要放大100倍到1.06Vrms。

**关键问题**：放大器的输入噪声会被放大100倍！

OPA2277的输入噪声密度：8nV/√Hz
在100kHz带宽内的总噪声：8nV × √100000 = 2.5μVrms
放大100倍后：250μVrms

信号10.6mVrms，噪声250μVrms → SNR = 32.5dB

THD测量的精度受限于SNR：
- 如果5次谐波幅度是基波的5%（THD=5%），谐波幅度=0.53mVrms
- 放大后=53mVrms，噪声=250μVrms → 谐波SNR=46dB → 够了
- 但如果THD=1%，谐波幅度=10.6μVrms → 放大后=1.06mVrms → 谐波SNR=12dB → 不够！

**结论**：30mV信号的THD测量精度受限于放大器噪声，低THD值测量困难。

**优化方案**：
1. 用更低噪声的运放（AD8065: 7nV/√Hz，改善有限）
2. 减小带宽（加低通滤波器，只保留到5次谐波=5kHz）
   - 带宽从100kHz降到5kHz → 噪声降低√20 ≈ 4.5倍
   - 但这只适用于基频1kHz的情况
3. 多次测量取平均（噪声降低√N倍）

### 量程切换的时机

题目要求30mV~600mV，跨度20倍。
- 量程1：直通（300mV~600mV）
- 量程2：×10放大（30mV~60mV）

**什么时候切换？**
不能在测量过程中切换（会引入瞬态），必须在测量前判断。

方案：先用量程1快速采样，如果信号太小（ADC值<200），切换到量程2重新采样。

## 三、TI MCU的选型陷阱

### MSP432P401R
- ADC: 14bit, 1MSPS
- 1MSPS对于100kHz信号的5次谐波(500kHz)刚好满足奈奎斯特
- 但没有余量！实际需要≥2.5MSPS才能保证精度
- **结论**：基本要求(1kHz)够用，发挥部分(100kHz)不够

### TMS320F28379D
- ADC: 16bit, 3.5MSPS
- 3.5MSPS对于500kHz有7倍过采样，精度好
- 但F28379D是DSP，开发环境(CCS)和编程方式与STM32差异大
- **结论**：性能最好，但学习成本高

### 我的建议
- 如果只做基本要求：用MSP432，简单够用
- 如果冲发挥部分：用F28379D，提前2周熟悉开发环境
- 折中方案：MSP432 + 外部ADC(AD9226)，但增加了硬件复杂度

## 四、10秒时间限制的优化

题目要求"一键启动后10秒内完成测量与显示"。

实际各步骤耗时：
| 步骤 | 耗时 | 优化空间 |
|------|------|----------|
| 粗测频率采样 | 10ms | 无 |
| 频率计算 | <1ms | 无 |
| 重配ADC | <1ms | 无 |
| 同步采样 | 10~50ms | 取决于频率 |
| FFT(1024点) | 3ms@168MHz | 用CMSIS-DSP库 |
| THD计算 | <1ms | 无 |
| OLED显示 | 50ms | 用DMA加速 |
| 蓝牙发送 | 100ms | 只发关键数据 |
| **总计** | **<220ms** | **远小于10s** |

时间完全不是问题。但如果用更大的FFT(4096点)或多次平均，时间会增加。
即使做10次平均，也只需要2秒，仍然在10秒以内。
