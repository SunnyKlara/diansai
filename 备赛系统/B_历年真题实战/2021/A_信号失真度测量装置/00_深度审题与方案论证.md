## 第一层：题目的精确解读

### 题目要求的精确数字

| 项目 | 基本要求 | 发挥部分 |
|------|----------|----------|
| 输入峰峰值 | 300mV~600mV | 30mV~600mV |
| 基频 | 1kHz | 1kHz~100kHz |
| THD范围 | 5%~50% | 5%~50% |
| 测量误差 | Δ=THDx-THDo≤5% | Δ≤3% |
| 谐波次数 | 到5次 | 到5次 |
| 时间限制 | ≤10秒 | ≤10秒 |
| MCU限制 | **必须TI MCU+片内ADC** | 同左 |
| 显示 | 显示THDx | +波形+归一化幅值+手机 |

### 隐含约束（题目没明说但必须遵守的）

**约束1**："除输入信号外，不得再有任何其他信号引入测量装置"
- 不能用外部参考信号
- 不能用外部触发信号
- 频率测量必须从信号本身提取

**约束2**："一键启动测量后，装置应在10秒钟内自动完成失真度测量与显示（期间不得有人工操作）"
- 不能手动调量程
- 不能手动设置频率
- 一切必须自动完成

**约束3**："参赛队必须自带本队自用的发生器参加赛区作品测试"
- 信号发生器的"谐波发生"功能是关键
- 赛前必须熟练操作信号发生器
- THDo（标称值）以信号发生器的设置值为准

**约束4**：误差定义 Δ = THDx - THDo
- 这是**绝对误差**，不是相对误差！
- THDo=10%时，THDx必须在5%~15%（基本）或7%~13%（发挥）
- THDo=50%时，THDx必须在45%~55%（基本）或47%~53%（发挥）
- 高THD比低THD更容易达标（因为绝对误差容限相同但信号更强）

### 评委会怎么测试？

根据题目描述，测试流程是：
1. 评委告诉你设置什么参数（频率、THD值）
2. 你在自带的信号发生器上设置
3. 按下装置的启动键
4. 10秒内装置显示THDx
5. 评委比较THDx和设置值THDo，计算Δ

**关键**：你需要在评委面前操作信号发生器！
如果操作不熟练，浪费时间，评委印象差。

## 第二层：误差预算（工程师视角的核心）

### THD测量的误差链

```
信号 → [信号调理] → [ADC] → [FFT] → [谐波提取] → [THD计算]
         ↓            ↓        ↓          ↓            ↓
       增益误差    量化噪声  频谱泄漏   峰值定位    计算精度
```

### 逐环节误差分析

#### 1. 信号调理误差
- 放大器增益误差：用0.1%电阻 → 增益误差<0.2%
- 放大器非线性：OPA2277 THD<0.0003% → 可忽略
- **结论**：信号调理引入的THD误差<0.01%，可忽略

#### 2. ADC量化噪声
- TI C2000 ADC：12bit，ENOB约10.5bit
- 量化噪声功率 = Vfs² / (12 × 2^(2×ENOB)) 
- 信噪比 SNR = 6.02 × ENOB + 1.76 = 65dB
- 量化噪声等效THD ≈ 10^(-65/20) = 0.056%
- **结论**：ADC量化噪声引入约0.06%的THD底噪

#### 3. 频谱泄漏误差
- **不做同步采样**：泄漏可导致幅值误差>10% → THD误差>5%
- **做同步采样**：泄漏接近零 → 误差<0.1%
- **做同步采样+平顶窗**：即使同步不完美，误差<0.01%
- **结论**：同步采样是必须的，平顶窗是保险

#### 4. 谐波峰值定位误差
- 同步采样后，谐波恰好在bin上 → 无定位误差
- 如果同步不完美，谐波可能偏移±1 bin → 取±1范围最大值补偿
- **结论**：取±1 bin最大值后，误差<0.5%

#### 5. 总误差预算

| 误差源 | 基本要求(1kHz) | 发挥部分(100kHz) |
|--------|---------------|-----------------|
| 信号调理 | <0.01% | <0.01% |
| ADC量化 | 0.06% | 0.06% |
| 频谱泄漏(同步+窗) | <0.1% | <0.5%* |
| 峰值定位 | <0.5% | <1.0%* |
| **总计(RSS)** | **<0.5%** | **<1.1%** |

*100kHz时误差更大，因为5次谐波=500kHz，接近ADC采样率的奈奎斯特极限。

**结论**：
- 基本要求Δ≤5%：总误差0.5% << 5%，**轻松达标**
- 发挥部分Δ≤3%：总误差1.1% < 3%，**可以达标但余量不大**
- 100kHz是瓶颈，需要足够高的采样率

## 第三层：MCU选型的深度论证

### TI MCU的选择空间

| MCU | ADC位数 | ADC速率 | DAC | FPU | 价格 | 适合度 |
|-----|---------|---------|-----|-----|------|--------|
| MSP430FR5994 | 12bit | 200kSPS | 无 | 无 | ¥80 | ✗ 速率太低 |
| MSP432P401R | 14bit | 1MSPS | 无 | 有 | ¥60 | △ 基本够 |
| TMS320F28379D | 16bit | 3.5MSPS | 有 | 有 | ¥150 | ✓ 最佳 |
| MSPM0G3507 | 12bit | 4MSPS | 无 | 无 | ¥60 | △ 速率够但无FPU |

### 为什么选MSP432而不是F28379D？

**表面上F28379D更好**（16bit ADC、3.5MSPS），但：

1. **F28379D的开发环境(CCS+C2000 SDK)学习曲线陡峭**
   - C2000的外设配置和STM32/MSP432完全不同
   - 如果赛前没有充分练习，赛场上可能卡在配置上
   
2. **MSP432的1MSPS对基本要求(1kHz)绰绰有余**
   - 1kHz的5次谐波=5kHz，1MSPS有200倍过采样
   - 14bit ADC的ENOB约12bit，SNR约74dB → 量化THD约0.02%
   
3. **发挥部分(100kHz)确实需要更高采样率**
   - 100kHz的5次谐波=500kHz，MSP432的1MSPS只有2倍过采样
   - 勉强满足奈奎斯特，但抗混叠滤波器要求很高
   - F28379D的3.5MSPS有7倍过采样，更从容

### 我的选择：MSP432P401R

**理由**：
1. 基本要求50分可以轻松拿满
2. 发挥部分的100kHz虽然紧张，但通过精心设计抗混叠滤波器可以应对
3. 开发环境(CCS+MSP432 SDK)比C2000简单得多
4. 赛场上的可靠性比极致性能更重要

**如果你的队伍有C2000经验，选F28379D更好。**

## 第五层：硬件架构

### 信号链路

```
BNC输入 → [保护电路] → [量程切换(CD4051)] → [放大/衰减] → [偏置+抗混叠] → ADC
                                                                              │
                                                                         MSP432
                                                                              │
                                                                    OLED + 蓝牙 + 蜂鸣器
```

### 量程设计

| 量程 | 输入范围 | 处理 | ADC输入范围 |
|------|----------|------|------------|
| 量程1 | 300mV~600mV | 直通(×1) | 300mV~600mV |
| 量程2 | 30mV~60mV | 放大×10 | 300mV~600mV |

自动量程切换：先用量程1采样，如果ADC峰值<200(约150mV)，切换到量程2。

### 抗混叠滤波器

MSP432 ADC采样率1MSPS，奈奎斯特频率500kHz。
需要在500kHz以上有足够衰减。

基本要求(1kHz)：5次谐波=5kHz，远低于500kHz，不需要特别的滤波器。
发挥部分(100kHz)：5次谐波=500kHz，恰好在奈奎斯特频率！

**这意味着发挥部分的抗混叠滤波器设计是关键。**

方案：二阶Sallen-Key低通，fc=400kHz
在500kHz处衰减约-6dB → 不够！
需要四阶（两级级联），在500kHz处衰减约-12dB → 勉强。

或者：降低目标频率到50kHz（5次谐波=250kHz），留更多余量。
但题目说"1kHz~100kHz"，不能缩水。

**这是这道题发挥部分的真正难点。**
