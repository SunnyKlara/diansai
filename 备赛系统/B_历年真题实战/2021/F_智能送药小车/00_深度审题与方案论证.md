# 2021 F 智能送药小车 —— 深度审题与方案论证

## 题目精确解读

| 项 | 要求 | 分值 |
|---|---|---|
| 基本(1) 送药 1/2 号病房 ≤ 20s | 12 |
| 基本(2) 送药中部病房（随机）≤ 20s | 18 |
| 基本(3) 送药远端病房 ≤ 20s | 20 |
| 发挥(1) 双车协同送同一病房 | 23 |
| 发挥(2) 双车协同送取不同病房 | 21 |

## 五层

### 1) 物理本质
**循迹 + 视觉识别 + 路径规划 + 双车协同**。  
属于控制 + 视觉的复合题，吃 OpenMV 的现成识别 + 经典 PID 循迹。

### 2) 数学层
- 循迹偏差 e（5 路红外）→ PD 控制：差速 = Kp·e + Kd·de
- 数字识别 → OpenMV K-Means + 模板匹配，0.3s/帧
- 路径规划 → 走廊节点图（药房 - 走廊 - 病房 = 树形）

### 3) 工程层
基本要求 50 分必须 ≤ 20s 到达：
- 走廊长 ~3m，病房 1~8 号最远 ~ 5m
- 平均速度 0.5 m/s 即可（含识别 + 减速 + 转弯）

**速度指标**：直线 0.8 m/s，转弯 0.3 m/s。

### 4) 双车协同（发挥）
两车不能同时在同一走廊（碰撞）→ NRF24L01 共享位置 + 互斥：
- 优先级仲裁（车 1 先走，车 2 让路）
- 简单状态机：占用 / 等待 / 离开

### 5) 评分
基本 50 + 报告 18 = 68（赛区一等奖底线）  
发挥 (1) 23 分相对易拿（同病房，时序错开），(2) 21 分难。

## 方案选择

| 方案 | MCU | 视觉 | 评分 |
|---|---|---|---|
| **MSPM0G3507 + OpenMV H7** | 主控+视觉分离 | 推荐（与 2024H 同款骨架）|
| STM32 + OpenMV | 同样可行 | 也行 |
| K210 单片 | 太复杂 | 不建议 |

**选 STM32F103 + OpenMV H7**（成本低，¥10 vs MSPM0 ¥40）。

## 控制架构

```
OpenMV H7 ──UART──→ STM32F103 ──TB6612──→ 4 电机
   ↑                    ↑
[病房号 1..8]      [5 路红外循迹]
                        ↓
                  [PID 差速 + 编码器]
                        ↓
                  [NRF24L01 双车通信]
```

| 编码器 | JGA25-370 11PPR |
| 轮径 | 67mm |
| 最高速度 | 0.8 m/s |
| 循迹采样 | 100 Hz |
| 视觉识别 | 0.3 s/帧 |

## 风险

| 风险 | 缓解 |
|---|---|
| 数字识别误判 | 多帧（3 帧）一致才确认 |
| T 路口冲过 | 路口前减速到 0.3 m/s |
| 双车碰撞 | NRF24L01 心跳 + 互斥锁 |
| 电池续航 | 2S 7.4V 2200mAh，单场 ≥ 30min |
