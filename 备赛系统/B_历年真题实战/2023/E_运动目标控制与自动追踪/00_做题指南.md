## 核心约束
**红色和绿色系统完全独立，不得有任何通信！**
绿色系统只能通过**视觉**获取红色光斑信息。

## 系统架构

```
┌─────────────────────┐     ┌─────────────────────┐
│  红色光斑控制系统     │     │  绿色光斑追踪系统     │
│                     │     │                     │
│  [STM32] ──→ [云台] │     │  [摄像头] ──→ [MCU] │
│              [红色激光]│     │  [MCU] ──→ [云台]  │
│                     │     │              [绿色激光]│
│  预设路径控制        │     │  视觉检测+PID追踪    │
└─────────────────────┘     └─────────────────────┘
        ↓                           ↓
    屏幕(白色,1m远)              屏幕(同一块)
    红色光斑                     绿色光斑追踪红色
```

## 红色光斑控制（基本要求核心）

### 路径预设
- 边线路径：正方形0.5m×0.5m的四条边
- A4靶纸路径：A4纸四周的胶带轮廓
- 需要精确的角度-位置映射

### 角度计算
```
屏幕距离D=1m，屏幕中心为原点
目标点(x,y)对应的云台角度：
  偏航角 = arctan(x / D) ≈ x/D (弧度，小角度近似)
  俯仰角 = arctan(y / D) ≈ y/D

例：x=0.25m → 偏航角 = arctan(0.25) = 14.0°
```

### A4靶纸旋转处理（基本要求4）
- 需要视觉识别A4靶纸的位置和旋转角度
- 或者手动输入旋转角度（按键设置）
- 根据旋转角度变换路径坐标

## 绿色光斑追踪（发挥部分核心）

### 视觉检测红色光斑
```python
# OpenMV检测红色光斑
RED_THRESHOLD = (30, 80, 40, 80, 10, 60)

img = sensor.snapshot()
blobs = img.find_blobs([RED_THRESHOLD], pixels_threshold=5)
if blobs:
    largest = max(blobs, key=lambda b: b.pixels())
    cx, cy = largest.cx(), largest.cy()
    # 发送给STM32
```

### PID追踪控制
```c
// 绿色云台PID
float error_x = target_cx - IMG_CENTER_X;  // 像素偏差
float error_y = target_cy - IMG_CENTER_Y;

float yaw_adj = pid_compute(&pid_yaw, error_x);
float pitch_adj = pid_compute(&pid_pitch, error_y);

servo_yaw += yaw_adj;
servo_pitch += pitch_adj;
```
