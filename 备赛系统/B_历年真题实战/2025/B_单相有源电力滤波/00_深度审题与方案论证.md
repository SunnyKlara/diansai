# 2025 B 单相有源电力滤波装置 —— 深度审题与方案论证

> 这是 2025 A 三相能量回馈、2023 A 单相逆变并联之后的电源类**第三胞胎**。  
> 核心矛盾：**测量 50 分（基本要求）拿到手软，APF 补偿 46 分（发挥）才是真考场**。  
> 评判维度：电源工程师、控制工程师、信号处理工程师、报告评审、安全审核员 五视角。

---

## 第一层：题目的精确解读

### 题目要求的精确数字

**基本要求（50 分，必须全拿）**

| 项 | 要求 |
|---|---|
| 非线性负载电路 | 4 组合（KD × KC：半波/全桥 × 无滤波/有滤波）|
| 输入电压 uS | 24 V AC 50 Hz（赛场隔离变压器提供）|
| 输出电流 IO | **≥ 2 A** |
| 谐波误差（对每次谐波幅值）| **≤ 3%**（对参考值的相对误差）|
| 测量项 | uS 波形、iL 波形、ILRMS、H₂~H₅ 谐波含有率 |

**发挥部分（46 分，主要竞争点）**

| 项 | 要求 | 分值 |
|---|---|---|
| iF 频率与 iS 主谐波一致 | 频率正确即可（看示波器）| 10 |
| KD/KC 断开下 HmS ≤ 5% | 半波/无滤波负载 → 主谐波抑制 | 10 |
| KD/KC 闭合下 HmS ≤ 5% | 全桥/有滤波负载（最难，电流脉冲极陡）| 10 |
| iS 整体 THD ≤ 5% | 综合考核 H₂..Hₙ 全谱 | 16 |
| **报告** | 八段式 | 20 |

### 隐含约束（题目没明说但必须遵守）

**约束 1**：**iS = iL − iF**（基尔霍夫节点电流定律）

电网电流 iS、负载电流 iL、APF 补偿电流 iF 三者**强耦合**：
- iL 是被动量（由非线性负载决定，APF 改不了）
- iS 是被动量（= iL − iF 节点约束）
- **iF 是控制量**（APF 唯一能动手的地方）

⇒ 控制目标：**让 iF 的瞬时波形 = iL 的谐波分量**，则 iS 自动变成纯正弦基波。

**约束 2**：iF 的频率必须包含 iL 的所有主谐波

iL 在桥式 + 滤波时主要含 1, 3, 5, 7 次（奇次）；半波时还有偶次。  
因此 APF 输出电流环带宽必须 **覆盖到 7 次 = 350 Hz**，闭环带宽建议 **≥ 1.5 kHz**（10 倍裕度）。

**约束 3**：HmS ≤ 5% 对每次 H 都成立，不是只看 THD

题目用了 "HmS"（specific harmonic content）= 单次谐波含有率。即使 THD 整体 5%，但 H₃ 单独 8% 也扣分。  
⇒ APF 必须**对每个谐波都有补偿能力**，不能"按平均值压"。

**约束 4**：负载切换时 APF 不能崩

KD/KC 是**赛场实测时切换的**——意味着 APF 必须能**在线适应负载变化**。基波提取算法不能依赖"预先标定"。  
⇒ 必须用**自适应**算法（滑动 DFT 或 dq 变换），不能用**离线 FIR 滤波器**。

**约束 5**：APF 不能反向污染 iS

如果 APF 的开关纹波（20 kHz）漏到电网，iS 反而 THD 升高。  
⇒ APF 输出端必须有 LC 滤波器（开关纹波衰减 ≥ 40 dB）。

### 评委会怎么测试？

测试流程（典型 2 小时）：

1. **基本(1)**：让选手展示 4 种 KD/KC 组合的非线性负载电路 → 8 分
2. **基本(2)**：示波器 / 装置自身 LCD 显示 uS 波形、iL 波形、ILRMS → 22 分
3. **基本(3)**：用频谱仪或装置 FFT 显示 H₂~H₅ → 评委对比标准值，误差 ≤ 3% 给 20 分
4. **发挥(1)**：开 APF，看 iF 波形 → 频率与 iL 主谐波（150 Hz / 250 Hz 等）一致就给 10 分
5. **发挥(2)(3)**：测 iS 的 HmS（每次谐波/基波）≤ 5% → 给 10+10
6. **发挥(4)**：测 iS THD ≤ 5% → 给 16

**关键洞察**：基本 50 + 报告 18 = **68 分（赛区一等奖线，必拿）**，发挥再拿 25 分就 93 分（国二上限 / 国一边缘）。

## 第二层：误差预算与算法精度

### 谐波测量精度 ≤ 3% 怎么保证

| 误差源 | 估算 | 控制 |
|---|---|---|
| ADC 量化噪声 | 12 bit / 2² = 0.024% | ✅ 远小于 3% |
| 频谱泄漏（非整周期采样）| 加汉宁窗 + 整周期采样 → < 0.5% | 必须做 |
| 互感器线性度（ZMCT103C）| ±1% @ 全量程 | ✅ |
| 调理电路偏置漂移 | 数字校零（开机自校） | < 0.3% |
| 量化截断 + sin 表精度 | 用 float32，sin 表 1024 点查表 | < 0.1% |
| **合计** | **< 2%** | **余量 1%** |

⇒ 算法上**必须**：
1. 整周期采样：fs / fund = 整数（200 点/周期 = 10 kHz / 50 Hz）
2. 加汉宁窗（窗校正 ×2.0）
3. 软件零点自校（开机时采 100 个点求平均，作为 offset）

### APF 电流跟踪精度 ≤ 5% HmS 怎么保证

控制环带宽分析：
```
负载 7 次谐波 = 350 Hz
开关频率 fsw = 20 kHz
LC 输出滤波器 fc = 1 / (2π√LC) → L=2mH, C=4.7μF → fc = 1.6 kHz
电流环带宽建议 = fc / 5 = 320 Hz ❌（不够覆盖 350 Hz）
```

⇒ **修正方案**：
- L 减到 1 mH → fc = 2.3 kHz → 电流环带宽 = 460 Hz ✅
- 或：fsw 升到 40 kHz → 滤波 fc 可以提到 3 kHz，环带宽 600 Hz ✅
- 选 **fsw = 20 kHz, L = 1 mH, C = 4.7μF**（开关损耗、电感铜损平衡）

### THD ≤ 5% 综合预算

```
iS_THD² = ΣHk² ≈ Hms_3² + Hms_5² + Hms_7²
       ≤ 3 × 5%² = 0.75%² → THD ≈ √0.75% ≈ 8.7%
```

⇒ 单次 5% 远不够，**实际每次必须压到 ≤ 3%** 才能保证总 THD ≤ 5%。这是评分(2)(3) 给 5% 但 (4) 给 5% 的**陷阱**——必须更狠。

## 第三层：MCU 与器件选型

### MCU：STM32F407 vs G474 vs F405

| 候选 | 主频 | FPU | 内存 | 双 ADC 同步 | 价格 | 评估 |
|---|---|---|---|---|---|---|
| STM32F407VET6 | 168 MHz | F32 | 192K | ✅ | ¥40 | ✅ 已选（2025 G 同款）|
| STM32G474RE | 170 MHz | F32+CORDIC | 128K | ✅ HRTIM | ¥45 | 性能更强但库少 |
| STM32F405RGT6 | 168 MHz | F32 | 192K | ✅ | ¥35 | F407 简化版，少 SDIO |
| GD32F407 | 168 MHz | F32 | 192K | ✅ | ¥30 | 国产替代，注意复位电路差异 |

**结论**：**STM32F407VET6**（与 2025 G 谐波分析机型一致，FFT 库可直接复用）。

### 关键器件选型

#### 1. 电流互感器 ZMCT103C（5A:5mA）

| 项 | 参数 |
|---|---|
| 变比 | 1000:1 |
| 频率响应 | 50 Hz~10 kHz（-3dB） |
| 线性度 | ±1% @ 5A |
| 相位误差 | < 0.5° @ 50Hz |

**关键**：互感器 -3dB 频率 = 10 kHz → 7 次谐波 (350 Hz) **完全在线性区**。如果用普通霍尔 ACS712，相位误差 5°+，**测谐波直接死**。

#### 2. ADC 调理电路（关键且复杂）

iL 经互感器 → 5 mA 电流 → 串 100 Ω 取样电阻 → ±0.5V 双极性电压。  
ADC 单极性 0~3.3V → 加偏置 1.65V，差分放大 ×2.5 → 0.4~2.9V → ADC 范围 12% 余量。

```
ZMCT103C(5mA) → R(100Ω, 1%) → +0.5V to -0.5V
       ↓
   OPA2277 加偏置(1.65V) + 增益2.5
       ↓
   0.4V~2.9V → ADC IN0
```

**关键**：取样电阻必须 1% 精度（影响绝对幅值校准），运放选 **OPA2277**（GBW 1MHz, 1uV/°C 漂移），不要用 LM358（漂移大、有死区）。

#### 3. APF H 桥 MOSFET：IRF3205 × 4

| 参数 | 值 | 是否够用 |
|---|---|---|
| Vds | 55 V | ✅ 母线 48V |
| Id | 110 A | ✅ 远超 2 A |
| Rds_on | 8 mΩ | ✅ 导通损耗忽略 |
| Qg | 146 nC | ⚠️ 偏大，开关损耗稍高 |
| 价格 | ¥2 | ✅ |

**冲刺方案**：Qg 大 → 开关损耗 ~ 0.8 W/管 × 4 = 3.2 W。换 IRFB7434（Qg=81nC）开关损耗减半，但 ¥6/个 × 4 = ¥24（成本翻 3 倍）。基础选 IRF3205。

#### 4. APF 输出电感（关键！）

L = 1 mH（详见第二层带宽分析）  
峰值电流 = 2 A × √2 + 50% 余量 = 4.2 A  
铁粉芯 T106-26：Al ≈ 93 nH/N², N = √(1e6 / 93) = **104 匝** of 0.8mm 漆包线  
饱和电流 ~ 3.5 A → **不够！**  
⇒ 升级到 **T130-26**（Al=200nH/N², N=70匝, 饱和 5A）✅

#### 5. LC 滤波器电容

C = 4.7 μF / 63V CBB（X2 安规电容，**禁用电解**——电解 ESR 大、寿命短、有极性）。  
fc = 2.3 kHz，对 20 kHz 开关纹波衰减 = 20·log(20/2.3)² = **38 dB** ✅

## 第四层：软件架构（重构方向）

### 现有代码的问题

1. ❌ **扁平结构**：只有 `apf_control.c` + `harmonic_analysis.c` 两个文件，没有 Core/Drivers/Algorithm 三层
2. ❌ **算法层依赖 HAL**：`apf_control.c` 顶部 `#include "stm32f4xx_hal.h"` —— 违反规范
3. ❌ **无 config.h**：参数（PI 系数、采样率、阈值）散落在两个 .c 文件
4. ❌ **缺主程序**：没有 `Core/main.c`，没有状态机
5. ❌ **缺驱动**：ADC、PWM、LCD、按键全部缺位

### 升级后的架构

```
01_代码/
├── config.h                       # 所有可调参数集中（关键）
├── Core/
│   └── main.c                     # 5 状态机：IDLE → MEAS_ONLY → APF_RUN → FAULT/STOP
├── Drivers/                       # 板级抽象层（HAL 仅在这里出现）
│   ├── adc_us_il.{c,h}            # 双通道 uS/iL 同步采样 + DMA + 软件零点自校
│   ├── pwm_apf.{c,h}              # H 桥互补 PWM 启停 + 死区设置
│   ├── lcd_tft.{c,h}              # 2.8" TFT 波形 + 数据显示（占位）
│   └── button.{c,h}               # 4 按键扫描（KD/KC/APF/MODE）
└── Algorithm/                     # 与硬件解耦（仅 stdint.h、math.h、float）
```

### 状态机

```
        ┌──────────┐   按键 KEY1
        │ ST_IDLE  │ ────────────→  采样校准 → 进入 MEAS_ONLY
        └──────────┘
              │ 错误
              ▼
       ┌──────────────┐                ┌─────────────┐
       │ ST_MEAS_ONLY │ ─APF_ON 按下──→│ ST_APF_RUN  │
       │ 测谐波/THD   │                │ APF 实时补偿│
       └──────────────┘ ←─APF_OFF────  └─────────────┘
              │  过流/欠压                      │
              ▼                                ▼
       ┌──────────────┐                 异常 ┌─────────────┐
       │  ST_FAULT    │ ←───────────────────│  保护动作   │
       └──────────────┘                      └─────────────┘
```

### 中断与时序

| 资源 | 周期 | 触发方式 | 用途 |
|---|---|---|---|
| TIM1 | 50 μs | UPDATE 中断 | 20 kHz APF 控制环（电流跟踪 PI）|
| ADC1+ADC2 | 50 μs | TIM1 触发 + DMA | 双通道同步采样 uS, iL, iF |
| TIM2 | 100 μs | UPDATE 中断 | 10 kHz 数据收集（FFT 用）|
| TIM3 | 200 ms | UPDATE 中断 | LCD 刷新 + 按键扫描 + RMS 计算 |

### 核心算法

#### 1. 滑动 DFT 提基波（apf_compensator）

```c
// N = 400 (= 20kHz / 50Hz), 一周期 400 点
a1 = (2/N) Σ iL[k] · sin(2πk/N)   // sin 分量
b1 = (2/N) Σ iL[k] · cos(2πk/N)   // cos 分量
iL_fund(t) = a1·sin(ωt) + b1·cos(ωt)   // 重构基波瞬时值
iF_ref = iL(t) - iL_fund(t)            // 谐波分量 = APF 参考
```

**优化**：滑动窗递推 → 每周期只算 1 次乘加，CPU 占用 < 5%。

#### 2. 电流跟踪 PI

```c
err = iF_ref - iF_act
integ += Ki·err·Ts
duty = 0.5 + Kp·err + integ      // ±0.5 偏置（H桥单极性 PWM）
duty = clamp(duty, 0.05, 0.95)   // 限幅防过调制
```

参数初值：Kp = 5.0，Ki = 500，Ts = 50 μs。**先开环验证波形跟随，再开 Ki**（防积分饱和振荡）。

#### 3. 1024 点 FFT 谐波（harmonic_fft）

```c
1. ADC × 1024 点（@ 10 kHz → 102.4 ms = 5.12 个工频周期）
2. 加汉宁窗 + 校正系数 ×2.0
3. arm_cfft_f32（CMSIS-DSP）
4. 取 |X[5×k]|，k=1..10 即 H₁ 到 H₁₀（5 = 50Hz × 1024 / 10kHz）
5. THD = √(ΣHk²) / H₁，k≥2
```

## 第五层：硬件难点

### 难点 1：母线 48V 预充电

母线电容 470μF×2 = 940μF。直接上电瞬间充电电流 = V·C/dt 可达数十 A。  
**对策**：加 50Ω 限流电阻 + 继电器并联（与 2023 A 同款方案）。软件 ST_IDLE → ST_PRECHARGE → ST_MEAS_ONLY，等 Vbus > 0.9·Vdc 才让继电器吸合。

### 难点 2：互感器小信号噪声

ZMCT103C 输出 5 mA → 100Ω 取样 = 0.5V 峰值 → 实际负载 1A 时只有 0.1V，**信噪比堪忧**。  
**对策**：
1. 取样电阻就近放（不通过长线传递大电流）
2. ADC 16 倍过采样（每次中断采 16 点取平均）
3. 软件中值滤波（窗 5）去除尖峰

### 难点 3：H 桥死区与开关损耗

死区太短（< 200ns）：上下管直通烧管。死区太长（> 1.5μs）：iF 输出畸变 → THD 升高。  
经验值：**500 ns**（IR2110 关闭延迟约 200ns，留足裕度）。

### 难点 4：APF 启动冲击

APF 启动瞬间，电感电流从 0 → iL 谐波幅值，如果直接全速 PWM，电流冲击 > 10 A，触发硬件保护。  
**对策**：软启动 ramp（1 秒内从 0% → 100% 系数），与 2025 A 完全相同的 ramp 思路。

### 难点 5：iF 极性反了 → 越补越差

软件极性写错（iF_ref = iL_fund − iL 而不是 iL − iL_fund），会让 iS 谐波翻倍。  
**对策**：
1. 上电后先用 50% 系数 ramp，**示波器观察 iS 谐波是否减少**
2. 若反而增加 → 立即切回 MEAS_ONLY，把 iF_ref 取负后再启 APF
