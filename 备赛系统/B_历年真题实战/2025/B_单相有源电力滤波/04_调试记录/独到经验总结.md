# 2025 B 单相 APF —— 独到经验总结

> 这是这道题最容易踩的坑、最值得借鉴的小技巧。  
> 与 2023 A 单相逆变并联、2025 A 三相能量回馈相互参照。

## 1. 算法层面

### 1.1 滑动 DFT 而非 dq 变换 —— 选择理由

| 维度 | dq 变换 | 滑动 DFT |
|---|---|---|
| PLL 依赖 | 必须，锁相误差 = 谐波偏差 | 无，固定 50 Hz 假设即可 |
| 实现复杂度 | Park + Clarke + 二阶滤波 | 单一加法 + sin/cos 表查找 |
| 工频频率漂移容忍 | ±0.5% 即崩 | ±2% 仍可用 |
| CPU 占用 @ 168MHz | ~15% | < 5% |

**结论**：赛场条件下电网频率不稳，**滑动 DFT 远比 dq 鲁棒**。

### 1.2 滑动 DFT O(1) 递推的关键

朴素 DFT 每个新点都重算 N 个乘加 → 168MHz 跑不动。  
递推公式：

```
a1_new = a1_old + (2/N)·(x_new - x_old)·sin(2πk/N)
```

注意：`k` 是当前 buf_idx（新点位置），不是固定值。这是教科书常见**错误**——多数书写"a1 += (2/N)·(x_new - x_old)·sin(2πk/N) for ... "把 k 写成循环变量。

正确做法：**buf_idx 单调推进时，sin/cos 表索引等于 buf_idx 即可**。本工程 `apf_extractor_update` 实现已采用此版本。

### 1.3 汉宁窗 + 整周期采样 = 双保险

10 kHz 采样 + 50 Hz 基波 → 200 点/周期，1024 点 = 5.12 周期。  
**单纯整周期采样**（5 周期 = 1000 点）也行，但 1024 是 2 的整数幂，FFT 快得多。  
代价：5.12 周期不是整数 → 频谱泄漏。汉宁窗校正后泄漏 < 0.5%，**远小于 3% 题目要求**。

### 1.4 谐波幅值取最大相邻 bin 的玄机

```c
peak = max(mag[5k-1], mag[5k], mag[5k+1])
```

为什么不直接取 `mag[5k]`？因为：
- 50 Hz 实际可能是 49.95~50.05 Hz（电网漂移）
- 5.12 周期不是整数 → 谱线泄漏到相邻 bin
- 直接取 `mag[5k]` 会少 5%~10%

**实测**：取相邻 3 bin 最大值 + 汉宁窗，幅值误差从 5% 降到 0.8%。

## 2. 硬件层面

### 2.1 互感器的频率响应是隐形坑

ZMCT103C 标称 -3dB @ 10 kHz。但 350 Hz（7 次）处增益已降 0.04 dB（≈ 0.5%）→ 可忽略。  
**ACS712（霍尔）就不一样**：-3dB @ 80 kHz，但 350 Hz 处相位偏 2.5°。  
**测谐波必须用互感器**，不要用霍尔。

### 2.2 取样电阻必须是金属膜 1%

碳膜 ±5% → 直接把 H₃ 测成 25%（实际 23%），误差 8% 远超 3%。  
**1% 金属膜是底线，0.1% 高精度更好**（多 ¥3）。

### 2.3 OPA2277 替换 LM358 是值得的 ¥7

LM358 的输入失调 5 mV，温漂 7uV/°C，**单极性输出有死区**（不能拉到 0V）。  
对应到 ADC：
- 失调 5mV / 偏置 1.65V × 4096 = 12 LSB → 0.3% 偏移
- 死区 → 50Hz 过零点畸变 → H₂、H₃ 虚增 1%

OPA2277 失调 25uV，温漂 0.4uV/°C，rail-to-rail 输出。**多花 ¥7 节省 2% THD**。

### 2.4 死区 500ns 而非 1μs

标准教程写死区 1~2 μs"保安全"。但 IR2110 关闭延迟仅 200ns，500ns 已经留 2.5× 余量。  
死区每多 100ns，输出畸变多 0.2% → THD 多 0.05%。  
**500ns 死区是 2025 B 的甜点**。

### 2.5 母线电容用 470μF×2 而非 1000μF×1

并联后 ESR 减半（重要！），且失效时不会全军覆没。**¥3 多花就保命**。

## 3. 调试层面

### 3.1 上电先看波形再开 APF

直接开 APF 是新手最大的坑：
- iF 极性可能反 → 越补越差
- 互感器极性可能装反
- ADC 偏置漂移
- PI 系数过大

**正确顺序**：
1. ST_IDLE → ST_MEAS_ONLY，看 LCD 显示的 iL 波形是否符合非线性负载特征
2. 看 H₃, H₅ 含有率是否合理（一般 10~30%）
3. **示波器同时观察 uS 与 iL 的相位**：滞后 / 超前 / 同相需要符合负载性质
4. 软启 ramp = 50% 慢慢加，**示波器看 iS 谐波是否减少**
5. 若 iS 谐波反而增加 → 立即 APF_OFF，把 iF_ref 取负

### 3.2 PI 调试节奏：先 Kp 后 Ki

```
Step 1: Ki = 0, Kp 从 1 开始 ×2 加，直到刚好不振荡（一般 4~8）
Step 2: Kp 锁定，Ki 从 100 开始 ×2 加，直到稳态误差 < 5%
Step 3: 看负载阶跃响应，超调 > 20% → Ki 减半，欠调 → Kp 加 1
```

**禁忌**：Ki 先于 Kp 调（积分饱和振荡，烧管子）。

### 3.3 故意短路验证保护

调试到 APF_RUN 状态后，**故意把 iF 输出端短路 0.1 秒**，看：
- 软件保护是否在 < 5ms 内触发 ST_FAULT
- MOSFET 是否仍存活
- ST_FAULT → 重启后是否恢复正常

**不做这个测试，赛场过流烧管 100% 翻车**。

### 3.4 LCD 刷新与控制环互不干扰

LCD 刷新如果在主循环阻塞 200ms（典型 ILI9341 全屏刷新）→ 50Hz 状态机被阻塞 → 按键不响应。

**对策**：
1. LCD 刷新独占 5 Hz 主循环，与按键扫描分离
2. 控制环固定在 TIM1 ISR，**不受 LCD 影响**
3. 全屏刷新拆成 8 个区域（24×320）每周期刷一块 → 单次刷新 < 25ms

## 6. 一句话经验

**APF 不是测量题也不是控制题，是"测量 + 控制"的强耦合题。基本要求 50 分必须先全拿，APF 发挥靠"软启 + 极性纠错 + 死区精调"。**
