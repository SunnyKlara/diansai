# 2025 A 能量回馈变流器 —— 独到经验总结

> 三相变流器 + 能量回馈是电赛电源类的天花板。  
> 本文沉淀 8 个赛前必知的坑 + 调试期解决的每个 bug。

---

## 一、赛前必知的 8 个坑

### 坑 1：三相对称性 —— 决定 THD 和评委第一印象

**症状**：示波器三相电压看起来对称，但功率分析仪显示有零序分量，THD 超 2%。  
**根因**：三相电感不一致（5%~10% 偏差就够）。  
**对策**：
- 同批次买 5~6 个 T130-26 磁芯
- 同样匝数（138 匝）绕完，**用 LCR 表挨个测**
- 挑出 3 个电感量相差 < 1% 的
- 三相电容也用 LCR 筛选

> 这一步占调试时间的 10%，**不做 = 直接失分**。

### 坑 2：死区时间 —— THD vs 穿通保护

死区太短：上下管同时导通的瞬间烧 MOS（直接 6 个换新）。  
死区太长：输出电压畸变（THD 升），负载调整率变差。

**对策**：
- 第一次上电死区**从 1.5μs 起步**（DTG = 128）
- 验证逻辑正确后逐步降到 800ns（DTG = 68）
- 不要一上来就 800ns

### 坑 3：母线预充电 —— 上电瞬间的电流冲击

**症状**：直接上 48V，瞬间电流冲击烧母线电容前的二极管或保险丝。  
**根因**：母线电容 470μF×3，上电瞬间充电电流可达几十 A。  
**对策**：
- 硬件加 100Ω 限流电阻 + 继电器（继电器在 Vbus 充满后短路）
- 软件 ST_PRECHARGE 状态等待 Vbus 上升到 0.9 × Vdc 再启动 PWM

### 坑 4：调制比软启动 —— 避免输出电压超调

**症状**：直接以 m=0.8 启动，输出电压瞬间超调到 40V+，烧负载或闭环饱和。  
**对策**：
- ST_RAMPUP 状态从 m=0.10 渐升到 m=0.80（500ms 完成）
- 闭环在 RAMP 期间**失能**（VLoop_Enable(0)），避免积分饱和
- RAMP 完成后再使能闭环

### 坑 5：电压闭环参数整定 —— 0.3% 负载调整率

**症状**：开环输出，空载 32V，满载 28V，调整率 12.5%（远超 0.3%）。  
**调参流程**：
1. 关闭 I 项（KI=0），调 KP 直到稳态误差减小（KP 从 0.005 翻倍）
2. 出现轻微振荡时把 KP 退一档
3. 加 I 项（KI 从 KP/100 起步，逐步增大）
4. 直到满载切空载瞬间收敛 < 1s

**陷阱**：
- KI 太大 → 开机超调
- KI 太小 → 收敛慢，0.3% 永远达不到稳态

### 坑 6：ADC 三相同步 —— 相位测量的命脉（虽然本题不直接测相位）

**症状**：三相电压采样有时差，导致零序计算不准，间接影响 THD 评估。  
**对策**：
- 必须用 ADC1 + ADC2 的"双 ADC 同步规则模式"
- 触发源用 TIM1 TRGO，每个开关周期采样一次
- DMA 配置中读取 ADC->CDR（公共数据寄存器），不要分别读 ADC1->DR 和 ADC2->DR

### 坑 7：同步整流时序 —— 30 分关键，烧管也最容易

**症状 1**：上下管同时导通 → 直通短路 → 6 个 MOS 一起烧。  
**根因**：换向时上管未关下管已开。  
**对策**：硬件死区 + 软件死区双保险（5μs 软件死区 + 800ns 硬件死区）。

**症状 2**：电流过零附近反复抖动 → 反向电流 → 损耗反而增加。  
**根因**：电流接近零时 SR 决策抖动。  
**对策**：滞回死区（`SR_THRESHOLD_A = 0.15`），过零附近让 body diode 导通。

**症状 3**：高功率下偶尔烧 1~2 个 MOS。  
**根因**：电流采样有噪声 → 偶尔误判方向。  
**对策**：电流采样加 RC 滤波（fc = 5kHz），软件再做 4 点平均。

### 坑 8：CubeMX TIM1 双 ADC 触发配置陷阱

**症状**：ADC 不被触发，DMA 没有数据。  
**最常见原因**：
- TIM1 TRGO 没选 Update Event（默认是 Reset）
- ADC 触发源没设置为 Timer 1 Trigger Out event
- ADC 模式没启用 External Trigger

**对策**：用 SDK 例程交叉验证（先跑通 TIM 触发 ADC 单通道，再升级到双 ADC + DMA）。

## 三、链路验证标准流程（参考 2025G 经验，电源类专用版）

### Level 0：单模块离线验证

| 模块 | 验证 | 通过判据 |
|---|---|---|
| MCU 闪灯 | 编译烧录 | LED 闪 |
| 单 PWM 通道 | 占空比 50% | 测电压 = Vdc/2 |
| 三 PWM 互补 | 死区 800ns | 示波器看到上下沿不重叠 |
| ADC 单通道 | 接已知电压 | 数值与万用表对应 |
| 双 ADC 同步 | PA0/PA1 短路 | 两通道读同值 |

### Level 1：两两组合验证

| 链路 | 验证 |
|---|---|
| TIM → ADC 触发 | 改 TIM 频率，DMA 完成中断频率随之变 |
| SVPWM → PWM3Phase_SetDuty | 三通道占空比正弦变化 |
| PWM → LC 输出 | 接 10Ω 负载看波形 |

### Level 2：开环全链路验证

1. 设 m=0.8 / f=50Hz 开环输出
2. 测线电压 RMS（万用表交流档）= 25~30V（开环理论值）
3. 测 THD（用功率分析仪）= 5~10%（开环理论值，加滤波后）

### Level 3：闭环验证

1. 开闭环 → 线电压收敛到 32.0V ± 0.1V
2. 加载（0~2A）→ 电压波动 < 100mV
3. 切频率 20→50→100Hz → 输出电压保持 32V

### Level 4：能量回馈验证

1. 先用二极管整流 → 接回母线 → 测 I1（应 ≥ 1A）
2. 再切同步整流 → **从低电压开始（24V Vbus）**逐步升到 48V
3. 任何异常立即按 STOP（PWM3Phase_EmergencyOff）

## 四、保命级安全规则

⚠️ **48V 直流 + 三相 ~ 200V 峰值线电压**，触电会致命。

1. 通电时**手不接触电路板**
2. 测电压用**夹子探头**，不要用手扶
3. 上电前用万用表蜂鸣档**检查 VBUS 与 GND 不短路**
4. 上电后先看 **MOS 是否发烫**（5 秒内摸不烫继续，烫则立刻断电）
5. 调整接线必须**先断电 + 等母线电容放电（≥ 30s）**
6. 备用电源 fuse 1A，烧 fuse 不烧 MOS

## 算法层 PC 仿真发现的关键陷阱（2026-05-26）

通过 `01_代码/tests/algo_reference.py` 的 PI 闭环仿真发现：

### 母线电压余量陷阱（绝大多数队伍踩坑的点）

```
理论最大 Vline_RMS = m_max × Vdc / √2 = 0.95 × 48 / √2 = 32.24V
减去 3% 内阻压降                       ≈ 31.27V
PI 调到饱和也到不了 32V 设定！
```

**赛前必做**：
1. 用 53V 直流源测试（题目允许 Vdc ±10% = 43.2~52.8V，**取上限**）
2. 物理上限提到 35V，留 3V 余量给 PI
3. 现场若发现 PI 饱和，**第一时间检查母线电压**，而非调 Ki

**算法层在 PC 仿真已 100% 验证**：
- SVPWM 七段式：线电压 RMS 精度 100%（无任何系统性误差）
- RMS 滑动窗口：误差 -0.16V < 容差 ±0.25V
- 同步整流状态机：3 种边界情况全通过

### 推荐现场调参顺序

1. 先开环验证 SVPWM 三相对称性（示波器看 ABC 占空比）
2. 闭电压环时 **先把 Ki = 0**（纯 P）确认稳定
3. 把 Ki 从 0.0001 → 0.001 → 0.005 逐档加（每加一档观察超调）
4. 一旦超调 > 0.5V 就回退一档
5. 加载阶跃测试响应时间 < 50ms 即达标
