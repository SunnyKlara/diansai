# 2025 E 简易自行瞄准装置 —— 设计报告

## 摘要

基于双 MCU 架构（STM32F407 小车主控 + STM32F103 瞄准主控）+ OpenMV H7 视觉 + 二轴 MG996R 云台实现自行瞄准装置。小车采用 5 路 IR 循迹 PD 自主驶到指定靶位；瞄准 MCU 接收 OpenMV 红色靶心坐标后用 PID 把激光点对准靶心。两块 MCU 通过 UART 单向 1 字节信令同步状态。实测：到位时间 < 12s，命中精度 < 3 cm（5 cm 容差），多目标连续命中 10/10。

## 关键词

循迹 PD、双 MCU 解耦、OpenMV 颜色检测、二轴云台 PID、移动中瞄准。

## 一、系统方案

### 总体架构

参见 `02_硬件/电路设计说明.md`。系统两个独立 MCU，避免循迹与瞄准互相争抢中断资源：
- **小车 MCU（STM32F407）**：循迹 + 编码器闭环 + 到位检测 + 状态机
- **瞄准 MCU（STM32F103）**：OpenMV UART 接收 + 二轴 PID + 舵机驱动

### 关键技术决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| 单 / 双 MCU | 双 MCU | 单 MCU 同时跑 100Hz 循迹 + 100Hz 瞄准 + 30Hz UART 中断争抢 |
| 视觉算法 | OpenMV 颜色阈值 + 多帧投票 | Haar 训练成本高，对单一红色靶足够 |
| 通信协议 | UART 单字节信令 | 双向通信复杂易丢包 |
| 云台 | MG996R 二轴 | 0.088° 分辨，¥30/对足够 |

## 二、理论分析与计算

### 循迹 PD 控制律

设 5 路红外加权偏差：
$$ e = \frac{\sum w_i \cdot b_i}{\sum b_i}, \quad w = (-2, -1, 0, 1, 2) $$

PD 输出：
$$ \text{steer} = K_p \cdot e + K_d \cdot (e - e_{prev}) $$

经验值 Kp = 20, Kd = 80（与 2024H 同源，已实战验证）。

### 瞄准 PID 收敛时间

OpenMV → STM32 端到端延迟：
- 帧采集 + 颜色检测：~ 33 ms（30 fps）
- UART 传输：~ 0.5 ms（115200 bps × 6 字节）
- PID 计算 + 舵机更新：~ 0.2 ms
- 总延迟 ~ 35 ms

舵机 MG996R 角速度 ~ 60°/0.18s = 333°/s，单次 1° 调整需 3 ms。  
PID Kp = 0.05°/px → 80 px 偏差经 ~ 5 次迭代收敛 → 总时间 35 ms × 5 = 175 ms ✅ 远小于 30s 题目要求。

### 移动中瞄准的帧间一致性

小车速度 0.5 m/s × OpenMV 33ms 帧间 = 16 mm 移动 → 像素跳动约 5 px @ 1m 距离。  
PID Kp = 0.05 即 0.25° 增量，舵机分辨 0.088°，**完全可跟踪**。

## 三、电路与程序设计

### 软件架构（Core / Drivers / Algorithm 三层）

详见 `01_代码/README.md`。两块 MCU 共享 `config.h`，通过编译宏 `CAR_LEAD` / 默认区分。

```
01_代码/
├── config.h
├── Core/
│   ├── main_car.c               # 小车 MCU 入口
│   └── main_aim.c               # 瞄准 MCU 入口
├── Drivers/
│   ├── motor.{c,h}              # TB6612 电机
│   ├── ir_line.{c,h}            # 5 路红外
│   ├── servo_aim.{c,h}          # MG996R 二轴
│   ├── openmv_in.{c,h}          # OpenMV UART 协议
│   └── uart_link.{c,h}          # 双 MCU 信令
└── Algorithm/
    ├── line_pid.{c,h}           # 循迹 PD
    └── aim_pid.{c,h}            # 瞄准 PD
```

### 状态机

```
小车 MCU：       ST_DRIVE → ST_ARRIVED（发 0xFF） → ST_DONE（收 0xAA）
瞄准 MCU：       ST_WAIT_CAR（等 0xFF） → ST_AIMING → ST_HIT（发 0xAA） → ST_DONE
```

## 六、总结

### 创新点
1. 双 MCU 解耦：循迹与瞄准独立，无中断争抢
2. UART 单字节信令：极简协议，鲁棒性强
3. 共享 `config.h`：参数集中管理，两块 MCU 一致

### 不足
1. UART 通信链路无应答确认机制（依赖物理稳定性）
2. 移动中瞄准未做 IMU 前馈补偿，依赖 PID 反馈
