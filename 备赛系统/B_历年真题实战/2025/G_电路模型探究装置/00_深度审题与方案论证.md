# 2025G 电路模型探究装置 —— 深度审题与方案论证

> 这份文档不是做题指南，而是一个工程师面对未知问题时的完整思考过程。

## 第一层：读懂题目在考什么

### 表面要求
- 基本要求：给定传递函数H(s)的RC有源滤波电路，探究装置产生信号使其按要求输出
- 发挥部分：对未知RLC电路自主学习建模，然后推理生成输出

### 深层考察
这道题考的不是某个具体技术，而是**工程方法论**：

1. **正向工程能力**（基本要求）：已知系统模型，设计输入使输出满足要求
   - 这考的是：你能不能用数学工具（传递函数）指导工程实践
   
2. **逆向工程能力**（发挥部分）：未知系统，通过测量建立模型
   - 这考的是：你能不能从数据中提取规律

3. **综合实现能力**：信号发生→采集→处理→输出，全链路打通
   - 这考的是：你能不能把理论变成实物

### 题目中的隐含约束（很多人会忽略）

**约束1**：探究装置与被测电路之间"输入、输出端均只有1根信号线和地线"
- 意味着不能在被测电路的输出端接反馈线回探究装置
- 基本要求(3)(4)中，探究装置必须**开环**控制被测电路的输出
- 不能做闭环！这大幅增加了难度

**约束2**：发挥部分(2)中，"信号发生器的输出为...正弦波、矩形波（占空比10%~50%）和其他周期信号"
- "其他周期信号"是什么？题目没说！
- 可能是三角波、锯齿波、甚至任意波形
- 你的方案必须对**任意周期信号**都能工作

**约束3**：发挥部分(1)学习时间≤2分钟，(2)启动后5秒内生成输出
- 2分钟学习：扫频测量必须高效
- 5秒输出：FFT处理必须实时

**约束4**：发挥部分(2)要求"波形无失真、在示波器上能连续同频稳定显示"
- "同频稳定显示"意味着输出信号的频率必须和输入信号完全一致
- 不能有任何频率偏移，否则示波器上波形会漂移
- 这要求输入和输出使用**同一个时钟源**

## 第二层：方案选择的深度论证

### 核心问题：如何"复现"一个未知电路的行为？

#### 方案A：参数化建模（拟合传递函数）

思路：
1. 扫频测量频率响应
2. 假设电路是二阶系统 H(s) = (as+b)/(cs²+ds+e)
3. 用最小二乘法拟合5个参数
4. 用H(s)计算任意输入的输出

优点：
- 模型紧凑（只有5个参数）
- 理论上对任意频率都准确（不只是测量过的频率）

缺点：
- **必须假设模型阶数**。题目说RLC各一个，最多二阶，但如果判断错了呢？
- **拟合算法在MCU上实现复杂**。非线性最小二乘需要迭代，可能不收敛
- **截止频率附近拟合精度差**。频率响应在截止频率处变化剧烈，几个测量点可能不够

#### 方案B：非参数化建模（频率响应查找表）

思路：
1. 扫频测量频率响应（增益+相位），存为查找表
2. 对输入信号FFT
3. 对每个频率分量，查表得到增益和相位
4. 应用增益和相位后IFFT

优点：
- **不需要假设模型结构**。测量多准就多准
- **实现简单**。FFT→查表→IFFT，都是标准操作
- **对任意周期信号都适用**。只要频率分量在测量范围内

缺点：
- **只在测量过的频率点上准确**。中间需要插值
- **测量点数有限**。2分钟内能测多少个频率点？
- **插值误差**。如果截止频率附近测量点不够密，插值会不准

#### 方案C：时域卷积（测量脉冲响应）

思路：
1. 向未知电路输入一个窄脉冲
2. 记录输出（=脉冲响应h(t)）
3. 对任意输入x(t)，输出y(t) = x(t) * h(t)（卷积）

优点：
- 一次测量就够（不需要扫频）
- 理论上包含所有频率信息

缺点：
- **脉冲响应的信噪比很差**。窄脉冲能量低，输出信号很弱
- **卷积计算量大**。N点卷积需要N²次乘法
- **脉冲宽度限制了带宽**。要覆盖50kHz，脉冲宽度需<10μs

### 我的选择：方案B（频率响应查找表），但做两个关键改进

**改进1：自适应扫频密度**
不是均匀扫频，而是先粗扫找到截止频率，再在截止频率附近加密。
这样用有限的测量点数获得最高的精度。

**改进2：相位连续性校验**
测量的相位可能有2π跳变（atan2的范围是[-π,π]）。
需要做相位解缠绕（phase unwrapping），确保相位曲线连续。
否则插值会在跳变点处产生巨大误差。

## 第三层：硬件架构的深度设计

### 信号链路分析

```
【学习阶段】
DDS(AD9833) → 未知电路输入 → 未知电路 → 未知电路输出
                    ↓                              ↓
              ADC_CH1(输入采样)              ADC_CH2(输出采样)
                    ↓                              ↓
                         STM32F407
                    FFT → 增益/相位计算 → 存入查找表

【推理阶段】
信号发生器 → 探究装置输入(高阻抗缓冲) → ADC采样
                                            ↓
                                     FFT → 查表 → 应用增益/相位 → IFFT
                                            ↓
                                     DAC输出 → 低通滤波 → 探究装置输出
```

### 为什么选STM32F407而不是其他MCU？

| 需求 | STM32F407 | MSP432 | C2000 |
|------|-----------|--------|-------|
| ADC速率 | 2.4MSPS ✓ | 1MSPS △ | 3.5MSPS ✓ |
| DAC | 12bit ×2 ✓ | 无 ✗ | 无 ✗ |
| 浮点运算 | FPU ✓ | FPU ✓ | FPU ✓ |
| FFT库 | CMSIS-DSP ✓ | 有 ✓ | 有 ✓ |
| DMA通道 | 16个 ✓ | 有限 △ | 有限 △ |
| 开发难度 | 低 ✓ | 中 | 高 |

**关键**：这道题需要DAC输出！MSP432和C2000没有片内DAC。
STM32F407有2个12bit DAC，可以直接输出合成信号。
这是选型的决定性因素。

### DDS选型：AD9833 vs AD9851

| 参数 | AD9833 | AD9851 |
|------|--------|--------|
| 最高频率 | 12.5MHz | 70MHz |
| 分辨率 | 28bit (0.1Hz) | 32bit |
| 输出幅度 | 0.6Vpp固定 | 可调 |
| 接口 | SPI | 并行/串行 |
| 价格 | ¥15 | ¥20 |

题目要求"最高频率不小于1MHz"。AD9833最高12.5MHz，足够。
AD9833的SPI接口更简单，输出幅度固定但可以外接VGA调节。

**选AD9833**。理由：够用、简单、便宜。

### ADC采样方案

**关键问题**：学习阶段需要同时采样输入和输出信号。

方案1：ADC1和ADC2双ADC同步模式
- 两个ADC由同一个定时器触发，同时采样
- 保证输入和输出的采样时刻完全一致
- **这是测量相位的前提！如果采样不同步，相位测量会有误差**

方案2：单ADC交替采样
- 一个ADC轮流采样两个通道
- 两个通道之间有时间偏移
- 对于低频信号（<10kHz）偏移可以忽略
- 但对于高频信号（50kHz），偏移会导致相位误差

**选方案1**。理由：相位精度是这道题的命脉。

### DAC输出方案

STM32F407的DAC：12bit，最高约1MSPS（DMA+定时器触发）

**问题**：1MSPS的DAC输出50kHz信号，每周期只有20个点。
波形会有明显的阶梯效应。

**解决**：DAC输出后加二阶有源低通滤波器
- 截止频率：100kHz（通过50kHz信号，滤除采样频率的镜像）
- 用OPA2277运放实现Sallen-Key拓扑
- 这样输出波形就是平滑的

**另一个问题**：DAC的输出范围是0~3.3V，但题目要求"峰峰值不低于1V"。
需要加一个偏置去除电路（去掉1.65V直流偏置）和放大电路。

### 实时处理的时序分析

推理阶段的处理链路：

```
ADC采集2048点 → FFT(2048) → 查表(1024个频率点) → IFFT(2048) → DAC输出2048点

时间预算（STM32F407 @168MHz）：
  ADC采集：2048 / 100kHz = 20.48ms（假设采样率100kHz）
  FFT：    2048点 ≈ 3ms（CMSIS-DSP库，硬件FPU）
  查表：   1024次插值 ≈ 1ms
  IFFT：   2048点 ≈ 3ms
  DAC输出：2048 / 100kHz = 20.48ms（与采集同步）
  
  处理时间(FFT+查表+IFFT) = 7ms
  采集/输出时间 = 20.48ms
  
  处理时间 < 采集时间 → 可以用双缓冲实现实时处理！
```

**双缓冲方案**：
```
时间线：
  |----采集buf_A----|----采集buf_B----|----采集buf_A----|
  |                 |----处理buf_A----|                 |----处理buf_B----|
  |                 |                 |----输出buf_A----|                 |
  
  采集和处理交替进行，DAC输出延迟一帧（20ms）
  对于50Hz的信号，20ms = 1个周期的延迟，可以接受
```

## 第五层：关键技术难点的逐一攻破

### 难点1：相位测量的精度

相位误差直接影响矩形波重建的质量。

**误差来源**：
1. ADC双通道采样的时间偏移 → 用双ADC同步模式消除
2. DFT的频谱泄漏 → 用同步采样（采集整数个周期）消除
3. 相位解缠绕错误 → 需要仔细处理

**相位解缠绕**：
```c
// atan2返回[-π, π]，相邻频率点的相位差可能超过π
// 需要加减2π使相位曲线连续

void phase_unwrap(float* phase, int N) {
    for (int i = 1; i < N; i++) {
        float diff = phase[i] - phase[i-1];
        while (diff > PI) { phase[i] -= 2*PI; diff -= 2*PI; }
        while (diff < -PI) { phase[i] += 2*PI; diff += 2*PI; }
    }
}
```

### 难点2：矩形波的处理

矩形波包含无穷多次谐波。50kHz的矩形波，其谐波频率为150kHz、250kHz...
这些频率超出了我们的测量范围（假设扫频到50kHz）。

**怎么办？**

对于超出测量范围的谐波，有两种处理方式：
1. **截断**：直接丢弃（增益设为0）→ 输出波形会有Gibbs现象（振铃）
2. **外推**：根据已知的频率响应趋势外推 → 更准确但有风险

**我的选择**：截断，但在报告中说明这是有意为之。
理由：未知电路是低通/高通/带通/带阻滤波器，高频谐波本来就会被衰减。
截断的效果和实际电路的效果是一致的。

### 难点3：DAC输出的同频同步

题目要求"在示波器上能连续同频稳定显示"。

如果输入信号来自外部信号发生器，其频率和我们的采样时钟不同步，
输出信号的频率会有微小偏差，导致示波器上波形缓慢漂移。

**解决方案**：
1. ADC采样时钟由STM32内部定时器产生
2. DAC输出时钟也由同一个定时器产生
3. 输入信号经过ADC采样→FFT→IFFT→DAC输出
4. 输出信号的频率由ADC采样率决定，和输入信号的真实频率无关
5. 但因为FFT→IFFT是线性操作，输出信号的频率成分和输入完全一致

**关键**：只要ADC和DAC使用同一个时钟，输出就和输入同频。
不需要PLL锁相，不需要频率检测。这是FFT方法的天然优势。
