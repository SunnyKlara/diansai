# 2025G 电路模型探究 —— 独到经验总结

## 一、这道题为什么是近五年的巅峰？

### 它考察的不是某个技术点，而是工程思维的完整闭环

```
测量 → 分析 → 建模 → 推理 → 验证
  ↑                              │
  └──────── 如果不准就迭代 ───────┘
```

这是工程师解决未知问题的标准流程。
其他题目考的是"你会不会做某件事"，
这道题考的是"你能不能搞清楚一个你不了解的东西然后复现它"。

### 它预示了电赛的未来方向

2025G的"自主学习"概念，和AI/机器学习的思路一脉相承：
- 训练阶段 = 扫频学习
- 推理阶段 = 频域处理生成输出
- 模型 = 频率响应查找表

未来可能出现更复杂的"学习"题目，比如非线性系统辨识。

## 二、最容易犯的三个致命错误

### 错误1：用传递函数拟合代替频率响应查找表

很多人的第一反应：测量频率响应 → 拟合H(s) = (as+b)/(cs²+ds+e) → 用H(s)计算输出

**问题**：
- 拟合需要选择模型阶数（二阶？三阶？）
- 拟合算法（最小二乘）在MCU上实现复杂
- 拟合结果可能不准（特别是在截止频率附近）
- 如果模型阶数选错，误差会很大

**正确做法**：直接用频率响应查找表+插值。
不需要知道电路是什么，只需要知道它在每个频率上的增益和相位。
这是**非参数方法**，比参数拟合更鲁棒。

### 错误2：忽略相位信息

有些人只测量增益（幅频特性），不测相位（相频特性）。

**后果**：正弦波输出幅度正确，但矩形波输出完全错误！

原因：矩形波 = 基波 + 3次谐波 + 5次谐波 + ...
每个谐波经过滤波器后不仅幅度变化，相位也变化。
如果只修改幅度不修改相位，各谐波的相位关系被破坏，
合成的波形和真实输出完全不同。

**教训**：增益和相位必须同时测量、同时应用。

### 错误3：扫频点数不够

如果只测20个频率点，在截止频率附近的增益变化可能被遗漏。
特别是高Q值的带通/带阻滤波器，峰值可能非常尖锐。

**解决**：在截止频率附近加密采样点。
先粗扫确定大致的截止频率，再在截止频率±20%范围内细扫。

## 三、矩形波处理的深层思考

### 为什么矩形波是这道题的试金石？

正弦波只有一个频率分量，只需要查一次频率响应表。
即使频率响应测量有误差，正弦波的输出也不会太离谱。

矩形波有无穷多个谐波分量（1, 3, 5, 7, 9, ...次），
每个分量都需要独立查表并应用增益和相位。
任何一个分量处理不对，输出波形就会明显失真。

### 矩形波的傅里叶级数

```
x(t) = (4/π) × [sin(ωt) + sin(3ωt)/3 + sin(5ωt)/5 + sin(7ωt)/7 + ...]
```

如果基频1kHz，占空比50%：
- 1次谐波：1kHz，幅度1.273
- 3次谐波：3kHz，幅度0.424
- 5次谐波：5kHz，幅度0.255
- 7次谐波：7kHz，幅度0.182
- ...
- 25次谐波：25kHz，幅度0.051

如果未知电路是截止频率5kHz的低通滤波器：
- 1kHz分量几乎不衰减
- 3kHz分量轻微衰减
- 5kHz分量衰减到0.707（-3dB）
- 7kHz分量大幅衰减
- 25kHz分量几乎完全衰减

输出波形：圆角矩形波（高频分量被滤掉）

**如果你的频率响应表在5kHz附近不够精确，输出波形的圆角程度就会不对。**

### 占空比不是50%的矩形波

题目说占空比10%~50%，步长5%。
占空比不是50%时，矩形波包含偶次谐波！

```
占空比D的矩形波傅里叶系数：
  cn = (2/nπ) × sin(nπD)
  
D=50%: c1=0.637, c2=0, c3=0.212, c4=0, c5=0.127 (只有奇次)
D=25%: c1=0.450, c2=0.318, c3=0.150, c4=0, c5=0.090 (有偶次！)
D=10%: c1=0.197, c2=0.187, c3=0.172, c4=0.151, c5=0.127 (几乎所有次都有)
```

**D=10%的矩形波几乎是脉冲信号，频谱非常宽！**
需要频率响应表覆盖到很高的频率才能准确重建。

## 四、DAC输出的关键问题

### 采样率和位数

STM32F4的DAC：12bit，最高约1MSPS（用DMA+定时器触发）

对于50kHz的信号，1MSPS只有20个采样点/周期。
这会导致输出波形有明显的阶梯效应。

**解决**：DAC输出后加低通滤波器（RC或有源），平滑阶梯。

### 实时性

题目要求"启动后5秒内生成信号输出"。

FFT(2048点) + 频域处理 + IFFT(2048点) ≈ 20ms
加上ADC采集和DAC输出的时间，总共约50ms。

但这只是处理一帧数据。信号是连续的，需要持续输出。

**方案**：双缓冲
- 缓冲区A正在DAC输出
- 同时缓冲区B在做FFT处理
- 处理完后交换A和B
- 实现无缝连续输出

## 五、2分钟学习时间的分配

| 步骤 | 时间 | 说明 |
|------|------|------|
| 粗扫(20个频率点) | 5秒 | 确定大致频率响应形状 |
| 判断滤波器类型 | <1秒 | 低通/高通/带通/带阻 |
| 确定截止频率范围 | <1秒 | 从粗扫数据中找 |
| 细扫(截止频率附近30个点) | 10秒 | 提高关键区域精度 |
| 全频段补充扫描(50个点) | 15秒 | 覆盖所有可能的谐波频率 |
| 数据存储和验证 | 5秒 | 存入查找表 |
| **总计** | **约36秒** | **远小于2分钟** |

剩余的时间可以用来做验证：
用已知信号（如1kHz正弦波）测试推理输出是否正确。

## 六、链路验证标准流程（最高标准刷题追加）

> 这一段是经验之谈：**永远不要等所有模块都做完才整体测试**。
> 每个模块单独验证 → 两两组合验证 → 全链路验证 → 闭环验证。

### Level 0：单模块离线验证（不接任何其他模块）

| 模块 | 验证手段 | 通过判据 |
|---|---|---|
| DDS | 示波器看 1kHz / 50kHz 输出 | 频率误差 < 0.1%、波形纯净 |
| ADC | 接 1kHz 已知信号 → 串口打印 → MATLAB 画图 | 看到正弦波、无毛刺 |
| DAC | 写正弦表 → 示波器看输出 | 平滑（带滤波器后） |
| OLED | 显示文字 | 显示正确 |
| 按键 | 串口打印按下事件 | 无重复触发 |

### Level 1：两两组合验证

| 链路 | 验证内容 |
|---|---|
| DDS → ADC | DDS 输出接 ADC，串口打印采样值，MATLAB 重建波形与 DDS 频率一致 |
| ADC → DAC（直通）| 关闭频率响应（H ≡ 1）→ ADC 采到什么 DAC 就输出什么 → 1:1 跟随 |
| DDS → 已知 RC 滤波器 → ADC | 单频测量增益与理论值对比 |

**关键校准点**：ADC → DAC 直通必须 1:1（输入幅度 = 输出幅度），这是后续一切的基础。

### Level 2：开环全链路验证（已知电路）

用已知 RC 低通滤波器（R=10kΩ, C=10nF, fc=1.59kHz）作为"被测电路"：

1. **学习阶段**
   - DDS 扫频 → ADC 采两路 → 计算频率响应
   - 与理论值对比（理论 fc=1.59kHz）
   - 通过判据：测量 fc 误差 < 5%

2. **正弦波推理**
   - 信号发生器输出 1kHz 正弦 → 装置输出 vs 已知电路输出
   - 通过判据：幅度误差 < 5%、相位误差 < 5°

3. **矩形波推理**
   - 信号发生器输出 1kHz / 50% 矩形 → 装置输出 vs 已知电路输出
   - 通过判据：示波器双踪，两个波形目视一致

### Level 3：未知电路验证（接近实战）

随便选一个未知 RLC 组合（不要事先计算理论值），重复 Level 2 的流程。  
通过判据：完全相同的工作流程、完全相同的精度要求。

### 失败模式诊断决策树

如果任何一级验证失败，按下面的决策树定位问题：

```
推理输出幅度不对？
  ├─ 关掉频率响应（H ≡ 1）→ 输出还是不对？
  │    ├─ 是：FFT 缩放错了 → 调 signal_processor.c 的 scale
  │    └─ 否：频率响应测量错了 → 进入"频率响应错误"分支
  │
  └─ 频率响应错误：
       ├─ 单频 DFT 用 MATLAB 离线对比 → 算法对不对？
       │    ├─ 对：硬件问题（ADC 偏置/噪声/同步）
       │    └─ 错：DFT 实现有 bug
       │
       └─ 增益对、相位错？
            ├─ 双 ADC 不同步 → 校准 ADC 偏移
            └─ 相位解缠绕错 → 串口打印相位曲线，肉眼检查跳变
```

### 验证流程的 commit 节奏

每完成一级验证就 commit 一次：

```
git commit -m "feat: DDS+ADC 直通链路 1:1 校准完成"
git commit -m "feat: 已知 RC 电路扫频学习与推理验证通过"
git commit -m "feat: 矩形波推理与已知电路一致"
```

这样万一后面验证失败，可以 `git diff` 看是哪一段引入的问题。
