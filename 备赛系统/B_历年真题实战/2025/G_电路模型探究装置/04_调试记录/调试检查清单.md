## 关键故障排查

| 现象 | 最可能的原因 | 不太可能的原因 | 解决 |
|------|-------------|---------------|------|
| 增益测量偏差大 | 放大器增益不准 | ADC精度问题 | 用万用表校准放大器增益 |
| 相位测量偏差大 | 双ADC不同步 | DFT计算错误 | 检查ADC多模式配置 |
| 推理输出幅度不对 | DAC缩放系数错误 | FFT归一化问题 | 用已知信号校准缩放系数 |
| 推理输出波形失真 | 相位解缠绕错误 | 插值精度不够 | 打印相位曲线检查跳变 |
| 矩形波输出不像 | 高频谐波被截断 | 正常现象 | 检查是否与实际电路一致 |
| 示波器上波形漂移 | ADC和DAC时钟不同步 | 信号发生器频率偏差 | 确保用同一个定时器 |
| 学习时间超过2分钟 | 扫频点数太多 | 等待时间太长 | 减少扫频点或减少等待周期 |

## 关键陷阱补充（最高标准刷题追加）

### 陷阱 A：CMSIS-DSP 增益归一化（最容易翻车的点）

CMSIS-DSP 的 `arm_cfft_f32` 在 **正变换不归一化**，**逆变换也不归一化**。  
这意味着：FFT → IFFT 一来一回会得到 **N 倍**于原信号的结果（N=2048 时是 2048 倍）。

`signal_processor.c` 中已经在最后乘了 `scale = 2048.0f / FFT_SIZE`，但这只能消除"FFT→IFFT 自身的 N 倍"——
真实信号链路里还有：
- ADC 量化（满量程 4096 = 12bit）
- 归一化系数（除 2048）
- DAC 量化（满量程 4096）
- 频率响应表中的 gain（线性增益，无单位）

**校准步骤**（必须做一次）：

- [ ] A.1 关闭频率响应（让 H(f) ≡ 1.0 + 0j），输入 1kHz / 1Vpp 正弦 → DAC 输出应该 ≈ 1Vpp
  - 实测 DAC 输出：______ Vpp
  - 与输入比值（应 ≈ 1.0）：______
- [ ] A.2 如果比值偏离 1.0 ，调整 `signal_processor.c` 里的 `scale` 直到 1:1
- [ ] A.3 校准完成后，再启用频率响应，验证未知电路输出是否一致

> **千万不要等到推理矩形波时才发现"幅度不对"** —— 那时候你不知道是 FFT 缩放错了还是频率响应测错了。
> 必须先单独把"FFT→直通→IFFT"链路校准到 1:1。

### 陷阱 B：PA4 引脚切换的死锁

学习阶段 PA4 是 SPI1_NSS（DDS FSYNC，GPIO 输出模式）；
推理阶段 PA4 是 DAC1_OUT1（模拟输入模式）。

切换需要重新 `HAL_GPIO_Init(GPIOA, &gpio)`，但**有几个坑**：

- [ ] B.1 切换前必须 `DDS_Stop()`（让 DDS 进入 RESET 状态，输出归零）
- [ ] B.2 切换时 PA4 短暂为高阻态，DAC 输出可能是不确定值
  - 解决：切换前后插 1ms 延时，允许 DAC 内部电压稳定
- [ ] B.3 推理结束、回到 IDLE 时，**必须把 PA4 切回 GPIO 输出模式**，否则下次学习时 DDS FSYNC 不工作
  - 在 `STATE_RUNNING → STATE_IDLE` 的转换处加上反向 GPIO 配置

```c
/* 推理结束时切回 SPI 模式（DDS FSYNC）*/
GPIO_InitTypeDef gpio = {0};
gpio.Pin   = DDS_FSYNC_PIN;
gpio.Mode  = GPIO_MODE_OUTPUT_PP;
gpio.Pull  = GPIO_NOPULL;
gpio.Speed = GPIO_SPEED_FREQ_HIGH;
HAL_GPIO_Init(DDS_FSYNC_PORT, &gpio);
HAL_GPIO_WritePin(DDS_FSYNC_PORT, DDS_FSYNC_PIN, GPIO_PIN_SET);
```

### 陷阱 C：DAC 输出在静默期的电压

DAC 上电默认输出 0V（DAC_DOR1 = 0）。  
推理刚启动时，DMA 第一次填充缓冲区前，DAC 输出可能是 0V 或者 1.65V（取决于初始化）。
这会导致**输出端有一个开机跳变**。

- [ ] C.1 DAC 初始化时把缓冲区填成 2048（中间值 = 0V 信号）
- [ ] C.2 检查 `dac_buffer[]` 是否在 `DAC_Out_Init()` 中初始化为 2048（已实现 ✓）
- [ ] C.3 推理停止时，软件主动写 2048 让 DAC 输出 0V，避免最后一帧定格在某个非零值

### 陷阱 D：低占空比矩形波的高频谐波

D=10% 矩形波的频谱几乎平坦延伸到非常高的频率。  
如果信号发生器输出 50kHz / 10% 矩形波，谐波可达 500kHz~1MHz。  
但我们的 ADC 采样率 200kHz，奈奎斯特 100kHz —— **超出 100kHz 的谐波会混叠回低频**！

- [ ] D.1 测试 D=10% 矩形波时，输入端**必须**有抗混叠滤波器（fc=80kHz 左右）
- [ ] D.2 如果观察到低频段有奇怪的"虚假谐波"，先检查抗混叠滤波器
- [ ] D.3 报告里说明：超出 100kHz 的谐波被截断（这是物理限制，与实际滤波器行为一致）

### 陷阱 E：相位测量的 ADC 偏移

双 ADC 同步模式下两个 ADC 的采样**几乎**同步，但同一通道内 ADC1 和 ADC2 的**采样时间** 有 1 个 ADC 时钟周期的偏移（约 30~80ns，取决于 ADC clock 配置）。

50kHz 信号下，30ns 对应相位误差 = 360° × 50kHz × 30ns = **0.54°**。  
如果你的 ADC clock 配置错了（比如 ADC clock = 21MHz，则 1 个周期 = 47ns）也只是 ~1°。

- [ ] E.1 校准方法：DDS 输出接到 ADC1 和 ADC2 的输入（短路二者）
  - 此时两个通道应该是 **完全同相位** 信号
  - 测量到的相位差就是系统偏移
  - 在 `freq_response.c` 中扣掉这个偏移
- [ ] E.2 实测系统相位偏移：______°
- [ ] E.3 校准后相位偏移应 < 0.5°
