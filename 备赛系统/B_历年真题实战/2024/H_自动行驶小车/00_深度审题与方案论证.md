# 2024 H 自动行驶小车 —— 深度审题与方案论证

> 这道题的核心难点不是循迹（弧线段有黑线），而是**直线段无引导线**的航迹推算精度。
> 评判维度：评委、工程师、老师、学生、架构师 五视角。
> 目标：满分 120，稳拿 80+，争取 100+。

---

## 第一层：题目的精确解读

### 题目要求的精确数字

| 项目 | 基本要求 | 发挥部分 |
|---|---|---|
| (1) A→B 直线 | ≤ 15 s, 20 分 | — |
| (2) A→B→弧→C→D→弧→A | ≤ 30 s, 20 分 | — |
| (3) A→C→弧→B→D→弧→A 交叉 | — | ≤ 40 s, 30 分 |
| (4) (3) 路径 4 圈 | — | 越快越好, 30 分 |
| 报告 | 20 分 | — |
| **MCU** | **必须 MSPM0 系列** | 同左 |
| 尺寸 | ≤ 25(L) × 15(W) × 15(H) cm | 同左 |
| 行驶 | **只能前进** | 同左 |
| 轮子 | 不能用履带、不能用麦轮 | 同左 |
| 引导 | 不能用摄像头；不能借助场地外环境 | 同左 |

### 场地几何（自己量出来的精确尺寸）

```
 A ◀═══════════════ 直线 100 cm ═══════════════▶ B
 ▲                                              │
 ║                                              ▼
 ║    弧线 DA                          弧线 BC  
 ║    R = 40 cm                        R = 40 cm
 ║    弧长 = π × 40 ≈ 125.66 cm        弧长 ≈ 125.66 cm
 ║                                              │
 D ◀═══════════════ 直线 100 cm ═══════════════▶ C

  对角线 AC = √(100² + 80²) = 128.06 cm
  对角线 AC 与 AB 夹角 = atan(80/100) = 38.66°
```

**注意**：题目原文是"A、B、C、D 四点构成长方形"，AB=CD=100cm，BC、DA 是连接的半圆弧线（半径=40cm，因此 AD=BC=80cm 是直径）。

### 隐含约束（题目没明说但必须遵守的）

**约束 1**："只能前进" → 所有路径段都不能倒车
- 路径(2) BC、DA 是半圆弧，转弯角度 180°，不能后退掉头
- 路径(3) 每个对角线终点要走半圆弧，转弯角度也是 180°（C→B 是逆时针半圆，B→D 又是新的半圆）

**约束 2**："不能借助场地外环境导航" → 不能用磁罗盘、不能用 GPS
- 但 IMU（加速度+陀螺仪）属于"自身姿态"，不属于"环境"，可以用
- 编码器、红外、超声波（朝下打地面）都可以用

**约束 3**：必须 MSPM0
- LaunchPad 自带 XDS110 调试器，不需要外置 J-Link
- 工具链：CCS Theia 或 IAR for ARM，**不是 Keil**
- 配置工具：SysConfig（不是 CubeMX，无图形化时钟树）

**约束 4**：尺寸 ≤ 25×15×15
- 常见智能车底盘 23×15 刚好卡边，要小心轮子凸出
- 高度 15 限制了不能堆太多模块（OLED + 电池板 + 主控）

### 评委会怎么测试？

按题目原文流程：
1. 评委指定从 A 点起步
2. 选手按下启动键
3. 计时器记录到达终点的时间
4. 检查路径是否正确（路径偏出场地标线 = 失败）

**关键观察**：
- 题目没规定"小车在每个角点必须精确停留"，只看**总时间 + 路径正确**
- 也就是说，**A→B 不必精确停在 B**，只要 B 处自然地拐入弧线即可
- 但"A→B 直线"任务 (1) 要求停在 B 点 —— **这个停车精度是 (1) 的关键**

## 第二层：误差预算（工程师视角的核心）

### 直线段（AB / CD / AC）的误差链

```
PWM → 电机 → 编码器 → 距离/速度估计 → 状态机判停
              ↓
          IMU(yaw) → 航向闭环 → 差速修正
```

### 逐环节误差分析

#### 1. 编码器分辨率

参数：JGA25-370（30:1 减速比，编码器 11 线，4 倍频后每圈 11×4×30 = 1320 PPR）  
轮子直径 D = 65 mm，周长 = π × 65 ≈ 204.2 mm  
分辨率 = 204.2 / 1320 ≈ 0.155 mm/脉冲 ✅

走 1 m 距离 = 4900 脉冲，量化误差 < 1 脉冲 = ±0.155 mm，**可忽略**。

> 如果用 JGA25-370 的 13 线版本（PPR=520），分辨率 = 0.39 mm/脉冲，仍然够。

#### 2. 左右轮差异（最大误差源）

实测同型号电机左右轮：
- 同 PWM 下转速差异 5%~15%（电机匹配差、齿轮间隙、轮胎磨损）
- 同 PWM 下加速曲线不一致（起步快慢不同）

**不做修正**：1m 直线偏移 50~150 mm，1.5m 对角线偏移 75~225 mm → **失败概率高**  
**只用速度环**：消除稳态差异，但起步段累计误差仍有 20~40 mm  
**速度环 + IMU 航向闭环**：累计偏差 < 10 mm ✅

#### 3. IMU yaw 误差

MPU6050 静态典型陀螺零偏 0.05°/s，温度漂移 0.01°/s/℃。  
1 m 直线 @ 0.5m/s = 2 s 行驶 → 漂移 0.1° → 横偏 = 1000 × tan(0.1°) = **1.7 mm**，可忽略。

但前提是 **每次上电做静态标定（200 次平均）**，否则零偏可能 0.5°/s+，1m 漂 17 mm。

#### 4. 起步打滑

起步瞬间 PWM 从 0 跳到工作值，电机扭矩冲击 → 轮子空转 → 编码器读数虚高。  
**对策**：加速度限幅（PWM 每 10ms 增加 ≤ 50），起步前 200ms 限速 30%。  
打滑残余误差 ≤ 5 mm。

#### 5. 弧线段循迹精度

5 路 TCRT5000 间距 = 1.5 cm，传感器横向覆盖 6 cm。  
弧线 R=40cm，理想轨迹偏差 ≤ ±3 cm 都能保持至少一个传感器在线。  
循迹 PID 控制下，弧线段累计偏差 ≤ ±5 cm（可接受，因为弧线终点连接直线段）。

#### 6. 任务 (1)(2)(3) 的总误差预算

| 任务 | 主要误差源 | 累计偏差 | 余量 |
|---|---|---|---|
| (1) A→B (1m) | 起步 + 左右差 + IMU 漂 | < 15 mm | B 点容差 50 mm，**余量 3×** |
| (2) 一圈 (≈ 5.51 m) | 上述 + 弧线段 ×2 + 段切换 | < 80 mm | A 点容差 100 mm，**余量 1.25×** |
| (3) 交叉 (≈ 5.62 m) | 上述 + 对角线 ×2（角度精度更难）| < 120 mm | A 点容差 100 mm，**余量 0.83×** ⚠️ |

**结论**：
- (1)(2) 稳拿 40 分，没悬念
- (3) 必须把 IMU 标定 + 起步打滑控制做到极致才能稳定通过
- (4) 4 圈累计误差会发散，需要每圈终点用弧线起点的红外传感器**重置航向**（弧线起点的传感器排列状态可作为"基准点"）

## 第三层：MCU 与器件选型的深度论证

### MSPM0 内部选哪一颗？

题目要求 "MSPM0 系列"，TI 官方有 G/L/C 三大系列：

| 型号 | 主频 | Flash | RAM | ADC | PWM | 价格 | LaunchPad |
|---|---|---|---|---|---|---|---|
| MSPM0L1306 | 32MHz | 64KB | 4KB | 1.5MSPS 12bit | 4 路 | ¥10 | LP-MSPM0L1306 |
| MSPM0G3507 | 80MHz | 128KB | 32KB | 4MSPS 12bit | 8 路 | ¥30 | **LP-MSPM0G3507** |
| MSPM0C1104 | 24MHz | 16KB | 1KB | 1MSPS 12bit | 2 路 | ¥3 | LP-MSPM0C1104 |

**选 MSPM0G3507**，理由：

1. **PWM 通道**：电机驱动 2 路 + 蜂鸣器 1 路，至少 3 路；G3507 的 8 路绰绰有余
2. **定时器编码器模式**：G 系列 TIMA0/TIMG6/TIMG7 支持编码器接口，L 系列也支持但通道数更紧
3. **RAM**：32KB 留给 IMU FIFO、循迹滤波、状态机栈足够；L 系列 4KB 紧张
4. **主频**：80MHz vs 32MHz，PID + IMU 解算实时性更宽裕
5. **生态**：LP-MSPM0G3507 是 TI 的旗舰评估板，参考例程最多

### 电机选型

| 型号 | 减速比 | 空载转速 | 编码器 | 电压 | 价格 |
|---|---|---|---|---|---|
| JGA25-370 12V | 30:1 | 333 RPM | 11 线 4 倍频 = 1320 PPR | 12V | ¥25 |
| JGA25-370 12V | 50:1 | 200 RPM | 同上 | 12V | ¥25 |
| MG310 12V | 35:1 | 350 RPM | 7 线 = 840 PPR | 12V | ¥35 |
| TT 马达 6V | 1:48 | 200 RPM | 无 | 6V | ¥10 |

**选 JGA25-370 30:1 12V**：

- 空载 333 RPM × π × 65 mm / 60 = **1.13 m/s 最高速度** → 1m 直线 < 1s 完成（远超 15s 要求）
- 30:1 减速比保证扭矩，起步不打滑
- 编码器一体化，不用额外接线
- TT 马达没编码器，**否决**
- 50:1 减速速度只 0.68 m/s，对 (4) 4 圈竞速不利

**实际工作点**：直线段 0.5 m/s（PWM ≈ 50%），弧线段 0.3 m/s（PWM ≈ 30%），(4) 段可以拉到 0.8 m/s。

### IMU 选型

| 型号 | 性能 | 接口 | DMP | 价格 |
|---|---|---|---|---|
| MPU6050 | 老牌、漂移略大、带 DMP | I2C | ✅ 板载姿态融合 | ¥8 |
| MPU9250 | 加磁力计 | I2C/SPI | ✅ | ¥25 |
| ICM-42688-P | 新一代低噪声 | I2C/SPI | 简化版 | ¥30 |
| BMI088 | 低噪声、稳定 | I2C/SPI | 无 | ¥35 |

**选 MPU6050**：

- 已有大量参考代码（移植成本低）
- 内置 DMP 可直接输出四元数 → yaw 角，省一套互补滤波代码
- 漂移在我们的"短时直线段"场景下足够
- 价格便宜，备件容易

**风险**：MPU6050 有山寨版（不带 DMP 固件），购买时认准"GY-521 红色 PCB"或"九成新原装"。

### 循迹传感器

5 路 TCRT5000 vs 8 路 vs 灰度模拟：

| 方案 | 精度 | 抗干扰 | 价格 | 适配性 |
|---|---|---|---|---|
| 5 路 TCRT5000 数字 | 中 | 强 | ¥8 | ✅ 弧线 R=40cm 够用 |
| 8 路 TCRT5000 数字 | 高 | 强 | ¥15 | 过度设计 |
| 5 路灰度 ADC | 高 | 中 | ¥10 | 弧线段无需如此高精度 |
| QTR-8RC | 高 | 强 | ¥40 | 过度设计 |

**选 5 路 TCRT5000 数字版**，理由：弧线段不需要亚毫米精度，5 路 + 加权位置 + 数字阈值就够。

### 电池

- 2S LiPo 7.4V 1500mAh 25C：电机额定 12V 但 7.4V 也能跑，扭矩降但够用，**且重量轻**
- 3S LiPo 11.1V：完美匹配 12V，但重量↑、电池保护↑、价格↑
- **选 2S 1500mAh** + LM2596 降压模块出 5V（给主控+OLED+IMU）

### BOM 总成本

详见 `02_硬件/材料清单_BOM.md`，预计 **¥210 左右**。

### 任务路径定义

```c

typedef enum { SEG_LINE, SEG_ARC } SegType;
typedef struct {
    SegType type;
    float   distance_mm;   // 直线段长度 / 弧线段弧长

// 任务 (1) A→B
const Segment task1[] = {
    { SEG_LINE, 1000.0f,    0.0f },   // A→B 直线 1m
    { 0 } // 哨兵
};

// 任务 (2) A→B→C→D→A
const Segment task2[] = {
    { SEG_LINE, 1000.0f,    0.0f },
    { SEG_ARC,  1256.6f,  180.0f },   // B→C 半圆顺时针
    { SEG_LINE, 1000.0f,    0.0f },
    { SEG_ARC,  1256.6f,  180.0f },   // D→A 半圆顺时针
    { 0 }
};

// 任务 (3) A→C→B→D→A
const Segment task3[] = {
    { SEG_LINE, 1280.6f,    0.0f },   // A→C 对角线
    { SEG_ARC,  1256.6f,  180.0f },   // C→B（外侧弧）
    { SEG_LINE, 1280.6f,    0.0f },   // B→D 对角线
    { SEG_ARC,  1256.6f,  180.0f },   // D→A（外侧弧）
    { 0 }
};
```

注意 (3) 的对角线长度 ≈ 128 cm，并非简单 √(100²+80²) —— 因为弧线起点不是 BC/DA 的端点，而是切线衔接点。**实战需要现场量**，代码里把它做成 `config.h` 可调常量。

### 时序

| 周期 | 任务 |
|---|---|
| **1ms** | SysTick 心跳；编码器中断累积（实际编码器中断由硬件触发，与 1ms 无关） |
| **10ms** | IMU 读取 + 速度环 PID（增量式）+ 航向环 PID |
| **20ms** | 红外读取 + 循迹环 PID + 状态机推进 |
| **100ms** | OLED 刷新（如有）+ 电压检测 |

10ms 选择理由：MPU6050 DMP 默认 200Hz 输出，10ms（100Hz）够用且不会拖累 PID 响应。

## 第五层：硬件难点的逐一攻破

### 难点 1：直线段不直 → IMU 航向闭环

**问题**：左右轮差异 5~15%，纯速度环 1m 直线偏 5~15cm。  
**方案**：
- 起步时记录当前 yaw 作为 setpoint
- 每 10ms 计算 yaw 误差 → PID → 输出"航向修正量"
- 修正量加到左轮 +X，右轮 -X，差速纠偏
- IMU 必须先做静态零偏标定，否则 setpoint 漂移

### 难点 2：弧线段进入/退出的衔接

**问题**：直线段速度 0.5 m/s 直接撞进弧线 → 离心 → 甩出黑线。  
**方案**：
- 直线段最后 100mm 减速到 0.3 m/s
- 弧线段进入瞬间检测"中央传感器从白→黑"作为弧线起点
- 弧线段保持 0.3 m/s
- 弧线退出条件：所有传感器都白 + 编码器走过预估弧长 90%（双条件 AND）

### 难点 3：(3) 对角线的角度精度

**问题**：A→C 对角线角度需精确，否则到达 C 时偏离弧线起点 → 弧线找不到黑线。  
**方案**：
- 起步前在 A 点执行"原地差速转 X 度"（X 由路径定义）
- 转动期间 IMU 闭环 → 转到 setpoint 后停下
- 然后开始直线段（航向 setpoint = 当前 yaw）
- 角度精度 ±0.5°，对角线 1.28m 横偏 ≈ 11mm，可接受

### 难点 4：(4) 4 圈累计误差

**问题**：单圈误差不大，但 4 圈 + IMU 长期漂移可能发散。  
**方案**：
- 每圈结束，弧线终点的传感器排列（例如 5 路全白）作为"基准信号"
- 检测到基准信号时，认为小车回到 A 点附近，**重置编码器累计距离**
- 不要重置 IMU yaw（因为 yaw 应该回到起点航向）
- 加入"离场地外阈值检测"：编码器 + IMU 推算位置超出场地 → 紧急停车

### 难点 5：MSPM0 工具链熟悉度
