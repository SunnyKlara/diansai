# 2024H 自动行驶小车 —— 做题指南

## 评分策略

| 项目 | 分值 | 难度 | 策略 |
|------|------|------|------|
| (1) A→B直线行驶≤15s | 20 | ★★ | 必拿 |
| (2) A→B→弧→C→D→弧→A一圈≤30s | 20 | ★★★ | 必拿 |
| (3) A→C→弧→B→D→弧→A交叉≤40s | 30 | ★★★★ | 争取 |
| (4) 按(3)路径4圈越快越好 | 30 | ★★★★ | 争取 |
| 报告 | 20 | ★★ | 必拿15+ |

**这道题的特点**：场地上只有两个半圆弧线，没有直线引导线！A→B、B→C等直线段需要**无引导线导航**。

## 场地分析

```
    A ─────────────────── B
    │                     │
    │    (无引导线区域)    │
    │                     │
    D ─────────────────── C
    
AB段: 直线100cm (无引导线)
BC段: 半圆弧R=40cm (有黑色弧线)
CD段: 直线100cm (无引导线)
DA段: 半圆弧R=40cm (有黑色弧线)
```

## 导航方案

### 方案A：编码器+IMU航迹推算（推荐）
- 弧线段：红外传感器循迹
- 直线段：编码器计距+IMU保持直线
- 转弯点：弧线结束时切换到直线模式

### 方案B：纯编码器里程计
- 预设每段路径的距离和转角
- 编码器精确计数
- 不需要IMU，但累积误差大

### 方案C：弧线检测+惯性导航
- 检测到弧线→循迹模式
- 离开弧线→直线模式（IMU保持航向）
- 最灵活但实现复杂

## 路径规划

### 要求(2)路径：A→B→弧→C→D→弧→A
```
A ──直线100cm──→ B
B ──右转沿弧线──→ C  (半圆弧，顺时针)
C ──直线100cm──→ D
D ──右转沿弧线──→ A  (半圆弧，顺时针)
```

### 要求(3)路径：A→C→弧→B→D→弧→A（交叉）
```
A ──对角线──→ C  (距离约141cm，角度约45°)
C ──左转沿弧线──→ B  (半圆弧，逆时针)
B ──对角线──→ D  (距离约141cm，角度约45°)
D ──左转沿弧线──→ A  (半圆弧，逆时针)
```

**交叉路径的难点**：对角线行驶需要精确的角度控制

## 关键约束
- **必须用MSPM0系列MCU**
- 小车≤25cm(长)×15cm(宽)×15cm(高)
- 只能前进不能后退
- 不能用履带/麦克纳姆轮
- 不能用摄像头
- 不能借助周围环境导航

## MSPM0开发注意
- 开发环境：CCS + SysConfig（不是CubeMX！）
- 定时器/PWM/GPIO配置方式与STM32不同
- 需要提前至少1周熟悉MSPM0开发流程
- LaunchPad自带调试器，不需要额外下载器
