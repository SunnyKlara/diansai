## 1. 系统方案论证

### 1.1 任务分析

题目要求自动行驶小车在指定矩形场地（A、B、C、D 四点，AB=CD=100cm，BC、DA 为 R=40cm 半圆弧线）按 4 种路径行驶，分别为 (1) A→B 直线、(2) A→B→C→D→A 一圈、(3) A→C→B→D→A 交叉、(4) (3) 路径 4 圈。  
**关键约束**：必须使用 MSPM0 系列 MCU；只能前进；尺寸 ≤ 25×15×15 cm；禁用摄像头与履带/麦轮；不能借助场地外环境导航。  
**核心难点**：直线段（AB / CD / AC / BD）**没有引导线**，必须通过航迹推算与航向闭环保持直线。

### 1.2 方案对比

针对"无引导线直线段"的导航策略，本组对比了三种方案：

| 方案 | 优点 | 缺点 | 评分 |
|---|---|---|---|
| A. 纯编码器里程计 | 实现简单 | 累计误差大，左右差异敏感 | 6/10 |
| **B. 编码器 + IMU 航向闭环** | 误差小，实现成熟 | 需 IMU 标定 | **9/10 ✅** |
| C. 弧线检测 + 惯性导航 | 灵活 | 状态切换复杂 | 7/10 |

选择方案 B：直线段用 IMU 锁航向，编码器累计距离判停；弧线段切换为红外循迹差速 PID。

### 1.3 系统组成

```
传感层      MPU6050(IMU) + 编码器×2 + 红外×5 + ADC 电压采样
             │
             ▼
控制层      MSPM0G3507（80MHz）
             │ ├─ 速度环（增量式，10ms）
             │ ├─ 航向环（位置式，10ms，直线段使用）
             │ └─ 循迹环（位置式，20ms，弧线段使用）
             ▼
执行层      TB6612 双 H 桥 → JGA25-370 减速电机×2
             │
             ▼
辅助层      OLED 显示 + 蜂鸣器 + 三色 LED + 4 按键
```

## 2. 理论分析与计算

### 2.1 控制环设计

**速度环**（增量式 PID）：  
`Δu(k) = Kp·(e(k)-e(k-1)) + Ki·e(k) + Kd·(e(k)-2e(k-1)+e(k-2))`  
适合无明确数学模型的电机系统，避免积分饱和。

**航向环**（位置式 PID）：  
`u(k) = Kp·e + Ki·∫e + Kd·de/dt`  
e = setpoint_yaw − measured_yaw（归一化到 [-180°, 180°]）  
输出作为左右轮速度差，通过差速实现转向修正。

### 2.2 误差预算（直线段 1 m）

| 误差源 | 量级 |
|---|---|
| 编码器分辨率 | < 0.2 mm |
| 左右轮不一致 | 5%~15%，由 IMU 闭环修正至 < 5 mm |
| IMU yaw 漂移 (2s @ 0.05°/s) | 横偏 < 2 mm |
| 起步打滑 | < 5 mm（PWM ramp 限制） |
| **累计** | **< 15 mm**，对 50 mm 容差有 3× 余量 |

### 2.3 关键参数推导

- 轮径 65mm，编码器 1320 PPR → 0.155 mm/脉冲  
- 直线段 PWM 500（≈ 50%）→ 速度 ≈ 0.55 m/s → A→B 1m 用时 < 2s  
- 弧线 R=40cm，弧长 = π·R = 125.7cm，弧线 PWM 350（≈ 0.4 m/s）→ 弧线段 ≈ 3.2s

### 2.4 任务时间估算

| 任务 | 路径长度 | 估计时间 | 题目限制 |
|---|---|---|---|
| (1) | 1 m | < 5 s | 15 s ✅ |
| (2) | ~5.5 m | < 15 s | 30 s ✅ |
| (3) | ~5.6 m | < 18 s | 40 s ✅ |
| (4) | ~22 m | < 70 s | 越快越好 |

## 3. 电路与程序设计

### 3.1 系统硬件框图

（占位：开赛后补 Visio 框图）

详见 `02_硬件/电路设计说明.md`。关键电源域：VBAT 7.4V → TB6612 VM；LM2596 7.4V→5V → 主控/IMU/OLED/红外；MSPM0 板载 LDO 5V→3.3V。

### 3.2 主要单元电路

- **电机驱动**：TB6612FNG，PWM 50kHz，方向 GPIO 切换
- **电源**：2S LiPo + LM2596 + 自恢复保险丝 + 总开关
- **IMU 接口**：MPU6050 I2C @ 400kHz，DMP 200Hz 输出
- **传感器**：5 路 TCRT5000 数字版，电位器调阈值
- **保护**：低压检测（ADC + 软件阈值）+ 堵转检测（PWM/编码器比对）

### 3.3 软件架构

代码组织（详见 `01_代码/README.md`）：
- **Drivers/**：板级抽象（MSPM0 driverlib），`motor / encoder / ir_sensor / imu_mpu6050 / button / led_buzzer`
- **Algorithm/**：与硬件解耦，`pid / path / line_drive / track`
- **Core/main.c**：状态机 + 时序调度
- **config.h**：所有可调参数集中

### 3.4 主要算法流程

主状态机（8 状态）：

```
ST_IDLE → (按 START) → ST_CALIB_IMU → 装载段
                                          ↓
       (循环：直/弧/转) ST_RUN_LINE / ST_RUN_ARC / ST_RUN_TURN
                                          ↓ 段完
                                      ST_SEG_DONE → 下一段
                                          ↓ SEG_END
                                       ST_DONE
       异常：ST_LOST（停车蜂鸣）
```

时序：
- 1ms SysTick 心跳
- 10ms 速度环 + 直线段航向 PID + 编码器采样
- 20ms 弧线段循迹 PID + 状态机推进 + 按键
- 100ms OLED 刷新 + 电池检测
