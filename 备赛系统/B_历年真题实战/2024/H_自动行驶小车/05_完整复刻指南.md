## 第 2 周：硬件搭建

### 2.1 焊接 / 组装顺序

**严格按此顺序，不要跳过！**

#### Day 1：底盘 + 电机
- 组装 4WD 亚克力底盘
- 4 个 JGA25-370 电机装好，**编码器引出线分清左右、A/B**
- 装 18650 电池盒
- **焊主开关**：电池正极 → 拨动开关 → TB6612 VM
- **检查**：万用表测开关 ON 时电池正负极通到 TB6612 VM/GND，OFF 时断开

#### Day 2：TB6612 模块 + 电机线
- TB6612 焊到底盘上（用 M3 铜柱）
- 4 个电机的 + - 线焊到 TB6612 AO1/AO2 / BO1/BO2
- **此时不要插核心板**

**测试 1**：手动用杜邦线短接 TB6612 AIN1 到 VCC（强制电机正转），看左轮是否转动。失败则查电机线。

#### Day 3：核心板 + 信号线
- LP-MSPM0G3507 用 M3 铜柱固定
- 按 `02_硬件/接线表.md` 第 2 节插所有 TB6612 信号线（M1~M7）
- 第 4 节插编码器（E1~E6）
- 第 6 节插 MPU6050（U1~U5）

**测试 2**：核心板上电（USB 给电），看 LED_PWR 亮，万用表测 3.3V 引脚。

#### Day 4：5 路 IR + 按键 + LED
- 5 路 IR 板装小车前部下方，距地 **1.0 cm**
- 按 `接线表.md` 第 5、7 节连完
- LED 加 1kΩ 限流电阻

**测试 3**：手指挡 IR 看板上 LED 切换。

#### Day 5：辅助电源
- LM2596 模块：调电位器到输出 5V（先空载调，再接负载）
- 接到 LP-MSPM0 5V 引脚

### 2.2 焊接完整检查（强制）

参照 `02_硬件/接线表.md` 第 8 节"接线检查表"逐条勾选：

- [ ] 电池极性正确

## 第 3 周：软件烧录

### 3.1 工程搭建

```
1. CCS → File → New → CCS Project
   选 MSPM0G3507, empty template
2. 加 SysConfig 文件
3. 按 01_代码/CubeMX_or_SysConfig 配置说明.md 配置完所有外设
4. Generate → 自动生成 ti_msp_dl_config.{c,h}
5. 把本仓库 01_代码/ 下：
   ├── config.h
   ├── Core/main.c
   ├── Drivers/*.c, *.h
   └── Algorithm/*.c, *.h
   全部 Add 到工程
6. Project → Build (Ctrl+B)
   预期：0 errors, 0 warnings
```

### 3.2 第一次烧录

```
USB 连接 LP-MSPM0 → Run → Debug (F11)
→ 等"Connected to Target"
→ Resume (F8)
```

预期现象：
- LED_R 闪烁（系统运行）
- 串口 (115200) 打印 "Hello MSPM0\r\n"
- 蜂鸣器响一声

### 3.3 模块单元测试

按本仓库提供的测试脚本逐个跑：

```bash
# 在 PC 端跑（验证算法逻辑无问题）
cd 01_代码/tests
python algo_reference.py
python cal_helper.py
```

预期：算法层 PC 端测试 0% 误差。

### 3.4 真机模块测试（**烧前 100% 跑过**）

按 `04_调试记录/调试检查清单.md` 逐项：

1. **电机测试**：串口发 `m 500 500\n` → 双轮正转。`m -500 -500` 反转。
2. **编码器测试**：手转轮子，串口打印 `enc_l = ?, enc_r = ?` 计数变化。
3. **IR 测试**：黑线下 5 路 IR 串口打印接近 0；白底接近 4095。
4. **MPU6050 测试**：旋转小车，yaw 读数变化。
5. **LED + 蜂鸣器测试**：系统状态切换时颜色对应。

## 第 4 周：调试出成绩

### 4.1 PD 整定

参照 `04_调试记录/独到经验总结.md` 第 1 节"PD 调参表"：

```
Step 1: KP_VEL=15, KI_VEL=0  → 单轮速度环不超调
Step 2: KP_VEL=15, KI_VEL=50 → 加积分消稳态误差
Step 3: 循迹 KP=18, KD=80    → 直道不抖
Step 4: 增 KD 直到弯道不丢线
```

### 4.2 三档速度方案（路径预判）

参照 `Algorithm/path.c`：
- 直道 0.8 m/s
- 微弯 0.5 m/s
- 急弯 0.3 m/s

### 4.3 上场前必做

- [ ] 现场重校 IR 阈值（**最容易翻车的一步**）
- [ ] 满电（电压 > 8.0V）
- [ ] IMU 静止校零 5 秒
- [ ] 跑 3 圈测平均时间，如 > 25s 则降速 / 升 KP
