## 一、赛前必知的 8 个坑

### 坑 1：220V 安全 — 不是技术，是态度

**血泪**：曾有队伍在调试时强弱电误接，烧穿主板还差点电到人。

**对策**：
- **永远先用 24V 变压器调试**，最后接 220V
- 强弱电板物理隔离（≥ 5cm）
- 调试时绝缘垫 + 单手原则
- 改强电接线 → 拔插头 → 等 30 秒 → 万用表测 0V → 才动手

### 坑 2：功耗 50mW 是硬门槛，不是软目标

**症状**：调试时不注意功耗，最后测发现 80mW，10 分直接没了。

**对策**：
- 选型时**第一时间**测功耗预算（不要等做完才测）
- 万用表 mA 档常驻电池正极
- 千万**不要用 OLED**（30mW 直接破预算）
- 千万**不要用 OPA2277**（5mW，是 OPA2333 的 50×）

### 坑 3：互感器变比误差远大于 ADC 精度

ZMPT101B 标称 1:1，实测可能 0.98:1 或 1.02:1（±2% 偏差）。

**对策**：
- **赛前必须校准**：用已知电压（可调电源）+ 万用表为基准，测装置读数 → 求校准系数 → 存 FRAM
- 至少 3 个校准点（50V / 100V / 220V）做分段线性插值
- 同样做电流校准（0A / 1A / 3A / 5A）

### 坑 4：50Hz 整周期采样 — 关键中的关键

**症状**：FFT 出来的谐波幅值偏低 5%~10%，THD 系统性偏小。

**根因**：fs / 50 不是整数 → 频谱泄漏。

**对策**：
- 采样率精确选 **10000 Hz**（不是 9999.5 Hz！）
- 采样点数 **1000 点**（不是 1024）
- 验证：50Hz 在 DFT 中 bin 必须是整数 5

### 坑 5：双通道 ADC 同步是 PF 测量的命脉

**症状**：纯阻负载 PF 应该 = 1，实测 0.95~0.98。

**根因**：电压电流采样有时间差 Δt → 等效相位差 = 2π × 50 × Δt。  
Δt = 100μs → 相位差 1.8° → PF = cos(1.8°) = 0.9995（OK）  
Δt = 1ms → 相位差 18° → PF = cos(18°) = 0.95（**已经破容差**）

**对策**：
- ADC 必须用"序列扫描"模式，两通道间时差 < 一个 ADC 时钟（30μs）
- 不能用两个独立 Timer 分别触发两个通道

### 坑 6：非线性负载下高次谐波信噪比低

**症状**：纯阻负载 THD 测量准；非线性负载（电脑电源）H7 / H9 偏差大。

**根因**：
- 高次谐波幅度小（H7 可能只有基波 5%）
- ADC 量化噪声相对放大
- 互感器在 500Hz 处带宽下降

**对策**：
- 累积多周期（5 周期 vs 1 周期）→ SNR + 7dB
- 选**带宽足够**的电流互感器（ZMCT103C 标称 50~10kHz）
- 加平顶窗作保险（即使整周期采样不严格）

### 坑 7：MSP430 浮点慢 — 用 IQmath 库或定点

**症状**：DFT 跑一次要 5ms 以上，CPU 占空比超预算。

**根因**：MSP430 没 FPU，软件浮点慢。

**对策**：
- 用 TI IQmath 库（Q15/Q31 定点）替代 float
- 只在最后一步显示时转 float
- 或者：sin/cos 用 LUT，不用运行时 sinf/cosf

### 坑 8：ZMPT101B 的副边电阻陷阱

ZMPT101B 内部已有副边电阻，**不能再外接太大的负载电阻**否则饱和畸变。

**对策**：
- 副边直接接运放输入（高阻抗），不要再加分压电阻
- 如需衰减，在副边输出接小电阻（< 10kΩ）后再分压

## 二、低功耗调试流程

### 第 1 步：测基线功耗

不烧任何用户代码，只跑 LaunchPad 出厂程序：
- 万用表 mA 档串入电池正极
- 实测电流 ≈ ____ mA（典型 0.3 mA）
- 功耗 = 3.3V × 该值 = ≈ ____ mW

### 第 2 步：分模块测增量

每加一个模块测一次：
- + ADC 持续运行 → 增量 ____ mA
- + DMA → 增量 ____ mA
- + 互感器副边接通 → 增量 ____ mA
- + LCD 显示 → 增量 ____ mA

> 哪一项超出预期，单独优化哪一项。

### 第 3 步：开 LPM 优化

- main 循环加 LPM3 → 总电流应降到 < 1 mA
- ADC 期间用 LPM0 → 进一步降低

### 第 4 步：常见省电技巧

| 优化 | 节省 |
|---|---|
| 关掉所有未用外设的时钟门控 | 0.5 mA |
| 把 GPIO 全部配为输出（拉低）| 0.1 mA |
| 把上拉电阻从 10k 改 100k | 0.1 mA |
| 主频 16MHz → 8MHz（DFT 仍能跑）| 1.2 mA |
| 用 IQmath 替代 float | 计算时间减半 → CPU 占空比减半 |

## 四、链路验证标准流程

### Level 0：单模块离线验证

| 模块 | 验证 | 通过判据 |
|---|---|---|
| MCU 闪灯 | 编译烧录 | LED 闪 |
| ADC 单通道 | 接 1.65V 直流 | 数值 ≈ 2048 |
| ADC 双通道 | A0/A1 短路 | 两通道读同值 |
| DMA | 启动 + 完成中断 | 进入回调 |
| LCD HT1621 | 显示 "8.8.8.8" | 全段显示 |
| 互感器 | 副边波形 | 示波器看到 220V × 1/100 ≈ 2.2V 波形 |

### Level 1：开环全链路（24V 模拟）

1. 24V/50Hz 变压器接入互感器一次侧
2. 软件计算 Vrms 应 ≈ 24V
3. 接已知电阻负载（10Ω）
4. Irms 应 ≈ 2.4A
5. P 应 ≈ 57.6W
6. PF 应 ≈ 1.0

### Level 2：闭环（220V 真实）

⚠️ 必须 Level 1 全过才能接 220V。

1. 加保险丝 + 双手离板
2. 上电瞬间观察 LED 状态
3. 测试纯阻负载 → 阻容 → 阻感 → 非线性

## 五、保命级安全规则（再强调一次）

⚠️ **220V 触电会致命**。

1. 通电时**手不接触电路板**
2. 测电压用**夹子探头**，不要用手扶
3. 上电前用万用表蜂鸣档**检查 VBUS 与 GND 不短路**
4. 上电前先看 **互感器极性是否反**（反向不会出事，但会有大共模噪声）
5. 调整接线必须**先断电 + 等母线电容放电（≥ 30s）**
6. 备用熔断器 1A，烧熔断器不烧主板
7. 最佳：**用 RCD 漏电保护插座**给整个测试系统供电

## 六、算法层金标准（PC 离线验证记录）

`01_代码/tests/algo_reference.py` 与 C 代码逻辑等价，先在 PC 上跑一遍验证理论精度，避免把"算法 bug"和"采样链 bug"混在一起。

### 验证结果（2026-05-26）

| 测试场景 | Vrms 误差 | Irms 误差 | P 误差 | PF 误差 | THD 误差 |
|---|---|---|---|---|---|
| 纯阻 220V/100Ω | 0.0000% | -0.0000% | -0.0000% | — | — |
| 阻感 PF=0.5 | 0.0000% | 0.0000% | 0.0000% | 0.0000% | — |
| 非线性 H1+H3+H5+H7+H9 | 0.0000% | -0.0000% | — | — | 0.0000% |

**结论**：

1. 整周期采样设计（fs=10kHz / N=1000 / FUND_BIN=5 整数）**完美消除频谱泄漏**
2. 算法理论误差远低于浮点精度
3. **1% / 2% 的容差预算全部可以留给硬件链路**（互感器 + 调理 + 校准）
