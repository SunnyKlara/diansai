# 算法层离线验证

`algo_reference.py` 是 `Algorithm/power_calc.c` + `Algorithm/dft_harmonic.c` 的 **Python 等价实现**，用 PC 上的 Python 3 直接跑：

```bash
python algo_reference.py
```

## 用途

1. **验证算法理论精度**：在合成信号（纯阻 / 阻感 / 非线性）上运行，确认设计本身是否能达到 1% / 2% 容差。
2. **MCU 移植后的金标准**：把 MSP430 代码采到的 `v_samples[1000]` / `i_samples[1000]` 通过 UART 导出 → 喂给本脚本 → 对比输出。两边差值应 < 1e-4，否则说明 C 代码移植有 bug。
3. **比赛现场快速排错**：硬件读数异常时，导出原始采样塞给本脚本。如果脚本输出正确，那 bug 在 MCU 端；脚本输出也错，那 bug 在采样链。

## 当前验证结果（已通过 ✅）

| 测试场景 | Vrms 误差 | Irms 误差 | P 误差 | PF 误差 | THD 误差 |
|---|---|---|---|---|---|
| 纯阻 220V/100Ω | 0.0000% | -0.0000% | -0.0000% | — | — |
| 阻感 PF=0.5 | 0.0000% | 0.0000% | 0.0000% | 0.0000% | — |
| 非线性（H1+H3+H5+H7+H9）| 0.0000% | -0.0000% | — | — | 0.0000% |

> 算法理论误差远低于浮点精度，**误差预算全部留给硬件链路**（互感器变比 + 调理 + 校准）。

## 关键约束

修改 `config.h` 后，必须同步修改本脚本顶部的常量定义：

```python
FUND_FREQ_HZ      = 50.0
CYCLES_PER_MEAS   = 5
SAMPLES_PER_CYCLE = 200
NUM_HARMONICS     = 10
```

否则金标准就失效了。
