# 2024 B 单相功率分析仪 —— 深度审题与方案论证

> **这道题的核心矛盾：高精度（误差 ≤ 1% / 2%）+ 极低功耗（≤ 50mW）+ 220V 市电安全**。  
> 三个目标互相牵制，国一队伍的胜负手在"既要又要又要"的工程平衡。  
> 评判维度：电力电子工程师、嵌入式工程师、报告评审、安全审核员、架构师 五视角。

---

## 第一层：题目的精确解读

### 题目要求的精确数字

| 项 | 要求 |
|---|---|
| 测量对象 | 单相 220V / 50Hz 市电及负载 |
| (1) 电压、电流 | 误差 ≤ 1% |
| (2) 有功功率 P、视在功率 S、功率因数 PF | 误差 ≤ 1% |
| (3) THD + 各次谐波（**至少 10 次**）| 误差 ≤ 2% |
| (4) 系统功耗 | **≤ 50 mW** |
| MCU | **必须 TI MCU** |

### 隐含约束（题目没明说但必须遵守）

**约束 1**：电气隔离  
"涉及 220V 市电"在题目里隐含——评测肯定接真实市电，**必须**电流互感器 + 电压互感器隔离。  
直接用电阻分压采样 220V 不仅危险，更会损坏 MCU。

**约束 2**：功耗 50mW 是"系统总功耗"  
包括 MCU、互感器副边、运放、显示、电源 LDO 自身静态电流等**所有**部件。
- 普通 OLED 显示 ~30mW → **直接破预算**，必须用段码 LCD 或电子墨水屏
- 普通运放 OPA2277 静态电流 800μA → 3.3V × 800μA × 2 = 5.3mW 已经占掉 10%
- ZMPT101B 自身约 6mW 是大头
- 算下来 MCU 留给自己的功耗预算只有 ~5mW

**约束 3**：谐波 10 次需要的采样率  
50Hz × 10 次谐波 = 500Hz。按奈奎斯特 ≥1kHz 即可，但实际为减小泄漏要 ≥4kHz。  
我们选 **10 kHz**（每个 50Hz 周期 200 点，整周期采样消除频谱泄漏）。

**约束 4**：测试负载多样性  
评测肯定包括：纯阻、阻容（PF 超前）、阻感（PF 滞后）、非线性（含谐波，比如电脑电源 / LED 灯）。
- 纯阻：PF = 1，THD = 0
- 非线性负载：PF < 1 且 THD 大，是测量精度的真考验

**约束 5**：要和功率分析仪（PA）对比  
评委会用专业 PA（如 Yokogawa WT300）作为基准，把你装置的读数和 PA 读数对比。
- PA 自身精度通常 0.1%，所以你的装置 1% 误差要相对 PA 算
- 即使你的算法理论精度 0.1%，互感器变比误差能达到 1~3% → **必须出厂校准**

### 评委会怎么测试？

1. 接标准 220V 市电
2. 接已知负载（电阻箱 / RC 串联 / RL 串联 / 真实非线性负载）
3. PA 和被测装置同时读数
4. 对比每项参数误差
5. 用万用表测电池电压 × 电流验功耗

**关键**：(4) 50mW 那 10 分是"是 / 否"题，没拿到就一分没有。

## 第二层：误差预算与功耗预算（工程师视角的核心）

### 测量误差链

```
220V 市电 → 互感器 → 调理 → ADC → 时域算法 / DFT → 显示
  ↓            ↓        ↓       ↓        ↓
 标准      变比 ~1%   增益噪声  量化  泄漏 / 截断误差
```

| 误差源 | 量级 | 处理 |
|---|---|---|
| ZMPT101B 变比 | ±1.5%（0~50°C 漂 ±2%）| **必须**校准 + 温补 |
| ZMCT103C 变比 | ±0.5%（穿心匝数可调）| 校准 + 出厂记录 |
| 100Ω 采样电阻精度 | 0.1% | 选 0.1% 精密电阻 |
| OPA2333 失调 | ±10μV | 信号 RMS 后自然抵消，可忽略 |
| ADC 量化（12bit）| 1/4096 ≈ 0.024% | 可忽略 |
| 频谱泄漏（同步采样后）| < 0.1% | 整周期采样 |
| 浮点累积误差 | < 0.01% | 用 double 累积 |
| **校准后合计** | **< 0.5%** | 1% 容差有 2× 余量 ✅ |

### 谐波误差预算

| 误差源 | 量级 |
|---|---|
| 高次谐波幅度本身小 → SNR 差 | 大 |
| ADC 量化噪声 → 高次谐波误差放大 | 中 |
| 50Hz 工频噪声耦合 → 偶次谐波（H2/H4...）|大 |

**对策**：
- 50Hz 整周期采样 + 提高 SNR：单周期 200 点 → 累积 5 周期 = 1000 点 DFT → SNR 提升 √5 ≈ 7dB
- 高次谐波低通滤波器（截止 1kHz）→ 抑制 ADC 后段噪声

### 功耗预算（核心拉分点）

设 3.3V 单电源（2 节 AA 1.5V + LDO 升压不实际，**用 3.3V CR2032 或 4.5V → 3.3V LDO**）：

| 模块 | 工作电流 | 功耗 |
|---|---|---|
| MSP430FR5994 @ 16MHz active | 2.5 mA → 30% 占空比 | 2.5 mW |
| MSP430FR5994 @ LPM3 standby | 0.5 μA → 70% 占空比 | 0.0012 mW |
| ADC12（采样时段） | 0.5 mA → 5% | 0.083 mW |
| OPA2333 × 2（恒开）| 17μA × 2 = 34μA | 0.11 mW |
| ZMPT101B（互感器副边） | ~ 2 mA | 6.6 mW |
| ZMCT103C × 1（电流互感器副边）| 0.05 mA | 0.17 mW |
| REF3030 基准 | 30μA | 0.1 mW |
| HT1621 LCD（含段码点亮）| 150μA | 0.5 mW |
| LDO TPS7A02 | 25 nA 静态 | < 0.001 mW |
| 偏置电阻（10kΩ × 2 串联给 1.65V）| 165μA | 0.5 mW |
| **合计** | **~ 5.5 mA** | **~ 11.6 mW** |

**11.6 mW < 50 mW**，**4.3× 余量**，但要严格防止误用普通 OLED / 普通运放破预算。

#### 如果不用低功耗设计的对照

| 对照（不优化）| 功耗 |
|---|---|
| MCU 持续 active（不进 LPM）| 8.3 mW |
| OPA2277 替代 OPA2333 | 5 mW |
| OLED 替代段码 LCD | 30 mW |
| **合计** | **~ 50 mW**（刚好破预算）|

**结论**：低功耗优化是国一关键。

## 第三层：MCU 与器件选型的深度论证

### MCU 选 MSP430FR5994 而不是 MSP432P401R？

| 项 | MSP430FR5994 | MSP432P401R |
|---|---|---|
| 主频 | 16 MHz | 48 MHz |
| ADC | 12bit / 200ksps | 14bit / 1Msps |
| 16MHz active 电流 | 2.5 mA | 7 mA |
| LPM3 standby 电流 | 0.5 μA | 0.85 μA |
| FRAM | 256KB | 256KB Flash |
| 浮点 FPU | 无 | 有 |
| 价格 | ¥80 | ¥60 |

**选 MSP430FR5994**：

- 功耗碾压 MSP432（同等占空比下省 60%）
- 16MHz 跑 200 点单频 DFT × 10 谐波 = 4000 次乘加 ≈ 0.5ms，绰绰有余
- FRAM 比 Flash 写入功耗低 100× → 长时间写入校准数据省功耗
- 12bit ADC 量化噪声 0.024% 远低于互感器变比误差，**精度不是瓶颈**
- 没有 FPU 但浮点软件库够用（用 IQmath 库甚至更快）

### ZMPT101B vs 直接电阻分压

| 方案 | 隔离 | 精度 | 功耗 | 安全 |
|---|---|---|---|---|
| ZMPT101B 电压互感器 | ✅ 完全隔离 | ±2% | 6.6 mW | ✅ |
| 电阻分压 + 隔离运放（ISO124）| 部分 | 0.1% | 30+ mW | ⚠️ |
| 电阻分压（无隔离）| ❌ 无 | 0.1% | 微小 | ❌ **危险** |

**选 ZMPT101B**：
- 安全是第一位（评委不会接受危险设计）
- 校准后精度可达 0.5% 以内
- 功耗预算下的最优方案

### 显示选段码 LCD（HT1621）

| 方案 | 功耗 | 显示 | 价格 |
|---|---|---|---|
| 0.96" OLED（SSD1306）| 20~40 mW | 高对比度，万能 | ¥10 |
| 1602 字符 LCD | 5~30 mW（带背光更高）| 16×2 字符 | ¥10 |
| HT1621 段码 LCD | 0.5 mW | 数字 + 单位标识 | ¥15 |
| 电子墨水屏（4.2"）| 0.01 mW（刷新瞬间 30mW）| 全图形 | ¥80 |

**选 HT1621**：
- 功耗最低（其他直接破 50mW 预算）
- 显示数字够用（V/I/P/PF/THD 各占一区）
- 价格便宜
- 段码屏的固定单位标识（"V"、"A"、"W"）省了字模空间

### 低功耗运放选 OPA2333

| 型号 | 静态电流 | 输入失调 | 噪声密度 | 价格 |
|---|---|---|---|---|
| OPA2277（标杆精度）| 800 μA × 2 = 1.6 mA | 10μV | 8 nV/√Hz | ¥8 |
| **OPA2333（推荐）** | **17 μA × 2 = 34 μA** | **10μV** | 55 nV/√Hz | ¥10 |
| TLV2333（同类）| 20 μA | 10μV | 60 nV/√Hz | ¥8 |
| LMV358（最便宜）| 100 μA × 2 | 3 mV ❌ | 39 nV/√Hz | ¥1 |

**选 OPA2333**：
- 静态电流仅 17μA → 50 倍优于 OPA2277
- 失调电压 10μV 与 OPA2277 相同（精度不打折）
- 噪声密度稍高但 50Hz 工频带宽下不影响 THD 测量

## 第四层：软件架构

### 三层结构

详见 spec design.md。核心是**算法层完全解耦**：

- `Algorithm/power_calc.c`：纯数学计算（输入电压/电流采样数组 + 缩放系数，输出 Vrms/Irms/P/PF）
- `Algorithm/dft_harmonic.c`：单频 DFT × 10 + THD（不依赖 CMSIS-DSP，自己实现 sin/cos LUT）
- `Drivers/`：MSP430 DriverLib 调用全部包在这里
- `Core/main.c`：状态机调度 + LPM3 进入 / 唤醒

### 状态机（低功耗专用设计）

```
ST_IDLE
  │ 1Hz 定时器中断唤醒
  ▼
ST_SAMPLING                 启动 ADC 双通道同步采样，1024 点（5 周期）
  │ DMA 完成中断
  ▼
ST_CALCULATING              时域 + DFT 计算（约 1 ms）
  │ 完成
  ▼
ST_DISPLAY                  HT1621 写显存
  │ 完成
  ▼
ST_LPM                      MCU 进入 LPM3，等下一秒唤醒
  │
  └── 总占空比 < 30%
```

### 时序

| 周期 | 任务 | 执行时长 |
|---|---|---|
| 1 s | 完整测量循环 | 总 < 100 ms |
| 0~5 ms | ADC 双通道采样 1024 点 | 5 ms（10kHz × 1024 / 1） wait, 102.4 ms |

**修正**：1024 点 / 10kHz = 102.4 ms。这部分 MCU 可以处于 LPM0（CPU 关 / ADC + DMA 跑），仍是低功耗。
- 0~102 ms：ADC + DMA 在跑，CPU 可在 LPM0（仅 ADC 模拟模块开），约 0.3 mA
- 102~103 ms：CPU 醒来做计算
- 103~105 ms：写 LCD
- 105 ms~1 s：CPU LPM3，约 0.5 μA

折合占空比：CPU active ~ 3 ms / 1000 ms = 0.3%，远低于预算的 30%。  
**功耗实际可能比预算的 13mW 还低**，给非线性负载下增加采样点数留余量。

### 关键算法：单频 DFT × 10 次

不用完整 FFT 因为：
- FFT(1024) 需要 5120 次乘加，5ms+
- 我们只关心 10 个频点（基波 50Hz + 9 次谐波）→ 单频 DFT 各 1024 次乘加
- 总共 10240 次 ≈ 1ms
- **省 80% 时间，省 80% 功耗**

实现上预先存 sin / cos LUT 避免运行时 sinf/cosf：
```c
// 50Hz 基波在 bin = 50 × 1024 / 10000 = 5.12 → round 到 5
// 但 5.12 不是整数 → 频谱泄漏
// 解决：调整采样率使 bin 落整数
// 5 周期 × 200 点 = 1000 点（不是 1024），调采样率 = 50Hz × 1000 / 5 = 10000 Hz ✓
// 所以实际用 N = 1000 点 DFT，基波在 bin = 50 × 1000 / 10000 = 5（精确整数 ✓）
```

## 第五层：硬件难点的逐一攻破

### 难点 1：220V 市电安全（不是技术，是态度）

- **不要**让强电与 MCU 信号地共用任何走线
- 互感器副边接 MCU，**强弱电完全隔离**
- 调试期用 **24V 变压器**模拟，**确认隔离可靠后**再接 220V
- 强电部分加保险丝（500mA 速断）+ 漆包铜线 ≥ 1mm²
- 板子分两块：**强电板 + 信号板**，物理隔开

### 难点 2：互感器变比校准

ZMPT101B 标称 1:1，但实测有 ±2% 偏差。

**方案**：
1. 用 PA 同时测装置和已知电压（可调电源）
2. 软件存"校准系数"（FRAM 永久保存）
3. 校准点：50V / 100V / 220V 三档分段线性插值

ZMCT103C 同样：
1. 穿心 N 匝（N = 10 时灵敏度高 10×，相当于 50A:5mA → 5A:0.5mA）
2. 用已知电流（电阻 + 电流表）校准

### 难点 3：50Hz 整周期采样

理论：N 个采样点 / fs = M 个 50Hz 周期，即 N / fs = M / 50 → fs = 50 × N / M。

实践：
- N = 1000，M = 5 → fs = 10000 Hz ✓
- N = 1024，M = 不能整除 → 必有泄漏
- **采样点数选 1000 而不是 1024**

### 难点 4：低功耗陷阱

- LDO 静态电流（普通 1mA → 用 TPS7A02 25nA 救命）
- 上拉电阻别太小（10kΩ vs 100kΩ：100μA → 10μA）
- 数字 IO 不用一律拉低或浮空（不要吊空）
- ADC 不采时进入"powerdown"
- 关闭所有未用外设的时钟（节省 100~500μA）

### 难点 5：非线性负载下的 THD 测量

非线性负载（如开关电源）的电流波形畸变严重，含丰富奇次 + 偶次谐波。

**精度难点**：
- 高次谐波幅度小 → 受 ADC 量化噪声影响大
- 50Hz 整周期不严格 → 频谱泄漏吞掉一部分基波能量
- 电流互感器在高次谐波处变比下降（带宽限制）

**对策**：
- 多周期累积（5 周期 vs 1 周期）→ SNR 提 7dB
- 选**带宽足够**的电流互感器（ZMCT103C 标称 50Hz~10kHz，10 次 = 500Hz 在带宽内）
- 软件加平顶窗作保险（即使采样不严格整周期）

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 220V 触电 | 极低（如严格隔离）| 致命 | 永远先在 24V 下调 |

| 互感器变比误差未校准 | 高 | 中 | 出发前完成校准 + 存 FRAM |
| 非线性负载 THD 偏差大 | 中 | 中 | 多周期累积 + 选合适互感器 |
| 段码 LCD 坏 | 低 | 高（无显示）| 备份 OLED（仅展示用，比赛时可换）|
