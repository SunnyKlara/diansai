# 如何贡献 / How to Contribute

> 欢迎补充真题、修正错误、贡献新的方向训练材料  
> 提交前请阅读本指南确保 PR 能被快速合入

---

## 仓库定位

本仓库是 **NUEDC（全国大学生电子设计竞赛）系统化备赛资产**。  
不是教程站、不是 demo 集合、也不是开源驱动库 —— 我们只收录"**比赛能直接用**"的内容。

**收录标准**：
- ✅ 真题深度审题、方案论证、复刻指南
- ✅ 已验证可编译的代码骨架（必须是真实 HAL/driverlib，不是伪代码）
- ✅ 经过实测的 PID 起步值、调试记录、踩坑经验
- ✅ 方向训练教程（按 8 大方向分类）
- ❌ 未验证的代码片段、网上抄来的"理论文章"
- ❌ 重复造轮子（已有 STM32G4 骨架的同类内容会被拒）

## 代码规范

### C 代码
- **缩进**：4 空格（不允许 Tab）
- **命名**：函数 `snake_case` / 宏 `UPPER_SNAKE` / 类型 `PascalCase`
- **配对**：每个 `.c` 配对 `.h`，`.h` 用 `#ifndef ... #define ... #endif` 防重包含
- **分层**：Drivers（HAL 调用）/ Algorithm（无 HAL 依赖）严格分离
- **配置**：所有可调参数（PID / 采样率 / 阈值）放 `config.h`
- **诊断**：getDiagnostics 必须 0 错误 0 警告

### Python 代码（tests/ 下）
- 与 C 代码一一对应（`algo_reference.py` 镜像 Algorithm 层）
- 使用纯 Python 实现（不依赖 numpy 高级 API），**便于嵌入式工程师对照**
- 包含 `run_validation()` 入口函数，无参数即可跑出全部数据

### Markdown 规范
- 首行必须是一级标题
- 表格用 Markdown 表格语法
- 代码块标注语言（```c / ```python / ```bash）
- 引用 IC 必须给出"完整型号 + 关键参数 + 数据手册 URL"
- BOM 表必须含：序号 / 名称 / 型号 / 参数 / 数量 / 用途 / 备注

## 提交规范

### Commit message
格式：`<类型>: <简述>`

| 类型 | 用途 |
|---|---|
| `feat` | 新增题目 / 新增功能模块 / 新增方向训练材料 |
| `fix` | 修代码错误 / 文档错误 |
| `docs` | 文档调整（不改代码逻辑）|
| `refactor` | 重构（不改外部行为）|
| `test` | 加测试 / 改 PC 端 Python 金标准 |
| `chore` | 工具配置 / 仓库管理 |

### 示例
```
feat: 2024 H 题循迹 PID 初版
docs: 补充 2021 A 题深度审题
fix: 2025G freq_response.c 算法层 HAL 污染修复
test: 2023A 补 PC 端 Python 金标准（spwm + control）
```

### 单次 commit 聚焦单一目的
不允许"杂烩 commit"：硬件 + 代码 + 文档全混在一起。

### 分支规则
- **不直接 push 到 main**
- 用 feature 分支：`feat/<年份>-<题号>-<简述>`
- 通过 PR 合入

## 提交前自检

```bash
# 1. 代码诊断 0 错误
# （在 IDE 里跑 getDiagnostics 或编译 build）

# 2. Python 测试通过
python "备赛系统/B_历年真题实战/<新题>/01_代码/tests/algo_reference.py"

# 3. 没有 secret 泄漏
# 检查 .env、密钥、个人证件信息

# 4. .gitignore 生效
git status --short  # 不应出现 *.apk、*.o、Debug/、.vs/ 等
```

## PR 模板

提交 PR 时填以下信息：

```
## 改动摘要
- [ ] 新增 / 修改 / 删除 文件清单：
- [ ] 影响范围：

## 自检清单
- [ ] 代码 0 错误（getDiagnostics 通过）
- [ ] Python 金标准跑通（如适用）
- [ ] 文档遵循 markdown 规范
- [ ] commit message 格式正确
- [ ] 不含敏感信息

## 测试 / 验证
- 测试方法：
- 测试结果：

## 关联 issue
Closes #__
```

## 不收录内容（避免徒劳）

❌ **未验证的代码** —— 哪怕看起来很专业，跑不通就不收  
❌ **盗版 / 灰色资源** —— APK、破解 Keil、私货数据手册  
❌ **个人信息 / 隐私** —— 身份证、私钥、未脱敏的设计文档  
❌ **重复内容** —— 已有材料覆盖的同类，请改为补充而非新建  
❌ **抄袭文章** —— 来源不明的"教程"
