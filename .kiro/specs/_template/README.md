# Spec 模板使用说明

## 目的

这是"最高标准刷题"流程的 spec 三件套蓝本。  
开新题时复制本目录，逐项替换占位符，节省审题阶段的初始化时间。

## 使用步骤

### 1. 复制目录

```cmd
xcopy /E /I ".kiro\specs\_template" ".kiro\specs\<年份>-<题号>-<题目名>"
```

例：  
```cmd
xcopy /E /I ".kiro\specs\_template" ".kiro\specs\2025-A-能量回馈变流器"
```

### 2. 替换占位符

打开三个文件 `requirements.md` / `design.md` / `tasks.md`，全文替换：

| 占位符 | 替换为 |
|---|---|
| `<年份>` | 2025 |
| `<题号>` | A |
| `<题目名>` | 能量回馈变流器 |
| `<…>` 其他 `<>` 包裹的占位 | 题目实际内容 |

### 3. 删除头部 frontmatter

复制后请删除三个文件顶部的 `inclusion: manual` 头（这是为了让模板本身不被 Kiro 自动注入）。

### 4. 创建工作目录

按 steering 规范在 `备赛系统/B_历年真题实战/<年份>/<题号>_<题目名>/` 下建：

```
00_做题指南.md
00_深度审题与方案论证.md
01_代码/  （Core/Drivers/Algorithm + README.md + config.h）
02_硬件/  （材料清单_BOM.md + 电路设计说明.md）
03_报告/  （设计报告.md + 测试数据模板.md）
04_调试记录/  （调试检查清单.md + 独到经验总结.md）
05_参考资料/  （可选）
```

### 5. 开新分支

按 steering 规范不能直接动 main：

```cmd
git checkout -b "feature/<年份>-<题号>-<题目名>"
```

### 6. 按 tasks.md 顺序推进

13 步严格不要跳，每个阶段完成立刻 commit。  
完成所有 19 个 task 后合 main。

## 与配套文档的关系

- **流程详解**：`备赛系统/E_赛场实战/最高标准刷题工作流.md`（13 步 + 8 维评分 + commit 节奏）
- **目录强制约定**：`.kiro/steering/电赛备赛仓库规范.md`
- **AI 协作工作流**：`备赛系统/C_通用能力/AI协作工作流.md`（6 阶段 + 反模式）
