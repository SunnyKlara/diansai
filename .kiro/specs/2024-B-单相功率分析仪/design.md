# Design — 2024 B 单相功率分析仪

## 总体方案

**架构**：220V 市电 → 互感器隔离 → 信号调理 → MSP430FR5994 双通道 ADC → 时域计算 (Vrms/Irms/P/PF) + 频域计算 (DFT 10 次谐波 + THD) → HT1621 段码 LCD 显示。

## 关键技术决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| MCU | MSP430FR5994 LaunchPad | TI 系唯一能在 50mW 限制内跑 16MHz + ADC + DMA 的 |
| 电流采样 | ZMCT103C 互感器 + 100Ω 采样电阻 | 5A/5mA 变比，安全隔离 |
| 电压采样 | ZMPT101B 互感器 | 250V/2mA，变比 1:1 隔离 |
| 调理放大器 | OPA2333（17μA 极低功耗）| 替代 OPA2277（800μA，超功耗预算）|
| 显示 | HT1621 段码 LCD | OLED 功耗 20~40mW 直接破预算 |
| 谐波算法 | **单频 DFT**（不是完整 FFT）| 只算需要的 10 个 bin，省 80% 计算量 |
| 同步采样 | TimerA 触发 ADC，每周期 200 点（基波 50Hz × 200 = 10kHz）| 整数倍周期消除频谱泄漏 |
| 低功耗 | LPM3 + 中断唤醒 | MCU 有效占空比 < 30% |

## 软件分层

```
01_代码/
├── config.h
├── Core/
│   └── main.c                 # 一键测量状态机 + LPM3 + LCD 刷新
├── Drivers/
│   ├── adc_dual.{c,h}         # MSP430 ADC12 双通道同步 + Timer 触发 + DMA
│   ├── lcd_ht1621.{c,h}       # 段码 LCD（极低功耗）
│   ├── button.{c,h}
│   └── led.{c,h}
└── Algorithm/
    ├── power_calc.{c,h}       # 时域 Vrms/Irms/P/PF
    └── dft_harmonic.{c,h}     # 单频 DFT × 10 谐波 + THD
```

## 状态机

```
ST_IDLE → ST_SAMPLING → ST_CALCULATING → ST_DISPLAY → ST_LPM → ST_IDLE
              ↑                                          ↓
              └─── 周期触发（1Hz 即可，刷新率不需要高）──┘
```

低功耗策略：90% 时间在 LPM3，10% 时间唤醒做"采样 + 计算 + 显示"。

## 误差预算

| 误差源 | 量级 | 处理 |
|---|---|---|
| 互感器变比误差 | 1~3% | 软件校准（出厂常数）|
| 采样电阻精度（0.1%）| 0.1% | 用精密电阻 |
| ADC 量化（12bit）| 0.025% | 可忽略 |
| 同步采样泄漏 | 0.1%（同步采样后）| 整周期采样 |
| **合计** | **< 0.5%（校准后）** | 满足 1% / 2% 要求 |

## 功耗预算（详见 BOM）

| 模块 | 功耗 |
|---|---|
| MSP430FR5994（@ 16MHz active 30%）| 2.5 mW |
| MSP430FR5994（@ LPM3 70%）| 0.07 mW |
| OPA2333 × 2 | 0.1 mW |
| ZMPT101B | 6.6 mW |
| HT1621 LCD | 0.5 mW |
| 其他 | 3 mW |
| **合计** | **~ 13 mW** |

50 mW 限制有 3.8× 余量。

## 兜底

| 风险 | 兜底 |
|---|---|
| 功耗超 50 mW | 降频到 8 MHz / 关掉串口调试 / 降低采样率 |
| 谐波到 10 次精度差 | 增加采样点数（256 → 512）|
| LCD 显示挤 | 多屏轮显（5s 切一次）|
| 现场 220V 危险 | 用 24V 变压器替代调试 |
