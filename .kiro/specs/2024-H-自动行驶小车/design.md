# Design — 2024 H 自动行驶小车

> 此文件描述本次"最高标准刷题"的整体技术决策。详细的论证（多方案对比、误差预算）放在 `备赛系统/B_历年真题实战/2024/H_自动行驶小车/00_深度审题与方案论证.md`。

## 总体方案：混合导航 = 弧线循迹 + 直线航推

直线段（AB/CD/AC/BD）无引导线 → **编码器里程 + IMU 航向闭环** 走直线。
弧线段（BC/DA）有黑线 → **5 路红外循迹 + 差速 PID**。
切换条件：检测到弧线传感器开始压线 / 编码器到达预定距离。

## 关键技术决策

| 决策项 | 选择 | 理由（详见深度审题） |
|---|---|---|
| MCU | MSPM0G3507 LaunchPad | 题目强制；4MSPS ADC、80MHz、自带 CAN/PWM |
| 电机 | JGA25-370 12V 带编码器 ×2 | 转速够、扭矩够、编码器一体 |
| 电机驱动 | TB6612 | 双路、效率高、PWM 控制简单 |
| 循迹传感器 | TCRT5000 5 路数字 | 弧线足够用、便宜、抗光好 |
| IMU | MPU6050（DMP）or ICM-42688 | 直线航向保持的命脉 |
| 反馈环 | 速度环（编码器，100Hz）+ 航向环（IMU，100Hz）+ 循迹环（红外，100Hz） | 三环串联 |
| 控制律 | 速度增量 PID、航向位置 PID、循迹位置 PID | 与现有 24/C 代码一致，可复用 |
| 路径切换 | 状态机：DRIVE_LINE → DETECT_ARC → TRACK_ARC → DETECT_END → DRIVE_LINE | 6 状态足够 |

## 状态机

```
        ┌─────────┐
   ┌──▶ │  IDLE   │
   │    └────┬────┘  按"启动键" + 选择任务号
   │         ▼
   │    ┌─────────┐
   │    │RUN_LINE │  按段定义直线，IMU 保持航向，编码器计距
   │    └────┬────┘  到达预定距离 OR 检测到弧线压线
   │         ▼
   │    ┌─────────┐
   │    │RUN_ARC  │  红外循迹差速 PID
   │    └────┬────┘  弧线结束（多个传感器同时白 + 编码器走过预估弧长）
   │         ▼
   │    ┌─────────┐
   │    │SEG_NEXT │  下一段切换
   │    └────┬────┘
   │         ▼
   │    ┌─────────┐
   │    │  DONE   │  蜂鸣器响 + 显示总耗时
   │    └────┬────┘
   └─────────┘
```

## 三环时序

| 周期 | 任务 |
|---|---|
| 1 ms | 心跳；编码器中断累积 |
| 10 ms | 速度环 PID（增量式）|
| 10 ms | 航向环 PID（直线段使用）|
| 20 ms | 循迹环 PID（弧线段使用）+ 状态机推进 |
| 100 ms | OLED 刷新 |

## 误差预算（直线段 1 m 行驶）

| 误差源 | 量级 | 处理 |
|---|---|---|
| 编码器分辨率 | 65mm·π / 11(PPR×4) ≈ 18.5mm/脉冲 → 1m 内 ±2 脉冲 ≈ ±37mm | OK |
| 左右轮直径不一致 | 1% 偏差 → 1m 偏 10mm（角度偏 0.6°）| IMU 闭环消除 |
| IMU yaw 漂移 | MPU6050 静态 0.1°/s → 1.5s 行驶漂 0.15° → 横偏 2.6mm | OK |
| 起步打滑 | 不可量化 | 加速度限幅 + 起步阶段降速 |
| **合计** | 横向偏差 ≤ 15mm，到达点误差 ≤ 5cm | **可达成** |

详细误差预算（含弧线段、交叉段对角线）放到 `00_深度审题与方案论证.md`。

## 兜底方案

- **IMU 失效**：退化为纯编码器里程，左右轮速度环保持，左右脉冲数差控制航向
- **循迹丢线**：保持上次差速方向修正 1 秒，仍丢则停车蜂鸣报警
- **任务 (3)(4) 太难**：先确保 (1)(2) 满分（共 40 分）+ 报告 18 分 = 58 分稳拿

## 风险

1. MSPM0 工具链熟悉度（CCS Theia + SysConfig）—— 调试期最大不确定性
2. IMU 标定（陀螺零偏）必须每次上电做，否则直线偏
3. 场地搭建（黑色弧线宽度、白色地面反射率）影响传感器阈值
4. 起步加速度过大导致打滑 → 编码器读数失真
