# Design — 2023 A 单相逆变器并联

> 与 2025 A 的关系：**同一架构的简化版**（单相 vs 三相、SPWM vs SVPWM、独立 vs 并联）。
> 大量算法层模块可在两题间互相复用：电压闭环 PI、RMS 计算、保护逻辑。

## 总体方案

**架构**：直流 36~48V → 单相全桥（4 × MOSFET）→ LC 滤波 → 24V/50Hz 交流输出。  
两台并联通过下垂控制（droop control）实现负载分配。

## 关键技术决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| MCU | STM32G474RE | HRTIM + 双 ADC，与 2025 A 同款 |
| 调制 | 双极性 SPWM（基本要求够用，简单可靠）| 单极性 SPWM 谐波更低，但需要更多 PWM 通道 |
| 全桥拓扑 | 互补 PWM + 死区 800ns | TIM1 自带互补，硬件死区 |
| 闭环 | 电压外环 PI（10ms 周期）+ 电流内环（可选） | 单环 PI 已能达到 0.2% 调整率 |
| 滤波器 | LC：L=1.5mH（T106-26 铁粉芯），C=4.7μF | fc ≈ 1.9kHz，对 20kHz 衰减 -41dB |
| 保护 | 软件过压过流（响应 100ms）+ 硬件过流比较器（响应 < 1μs） | 双重保险 |

## 软件分层

```
01_代码/
├── config.h                    # 所有可调参数集中
├── Core/
│   └── main.c                  # 主循环 + ADC/PWM 中断
├── Drivers/                    # 板级抽象（HAL）
│   ├── pwm_full_bridge.{c,h}   # TIM1 互补 PWM 启停（新建，封装裸 TIM 操作）
│   ├── adc_sample.{c,h}        # ADC + DMA 双通道
│   ├── display.{c,h}           # SSD1306 OLED
│   └── protection.{c,h}        # 软件保护
└── Algorithm/                  # 与硬件解耦
    ├── spwm.{c,h}              # SPWM 正弦表 + 角度推进（去 HAL 依赖）
    └── control.{c,h}           # 电压闭环 PI（去 HAL 依赖）
```

## 与 2025 A 共享的资产

- `Algorithm/control.c`：与 2025 A `voltage_loop.c` 算法等价（PI + 抗饱和 + 限幅）
- `Drivers/adc_sample.c`：与 2025 A `adc_3phase.c` 是同一思路（DMA + 触发同步）
- `Drivers/protection.c`：故障状态机可作为通用模板

## 误差预算

| 误差源 | 量级 |
|---|---|
| SPWM 谐波 → LC 衰减后 | < 1.5%（满足 2% 容差）|
| 死区 800ns / 50us 周期 | 1.6% 占空比畸变 → 闭环消除 |
| 母线电压波动 → 输出 | 闭环抑制 |
| **闭环后总误差（负载调整率）** | **< 0.1%（达标）** |

## 风险

| 风险 | 缓解 |
|---|---|
| 死区导致死区畸变 → THD 失败 | 软件死区补偿（不易，先试 800ns + 闭环）|
| 母线电容预充电缺失 | 加 100Ω 限流 + 继电器 |
| 双机并联振荡 | 下垂特性 + 同步 PLL（基本要求不用，发挥 (2)(3) 必要）|
